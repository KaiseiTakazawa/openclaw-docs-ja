# ダッシュボード

# Dashboard (Control UI)

Gatewayダッシュボードは、デフォルトで`/`で提供されるブラウザのControl UIです
（`gateway.controlUi.basePath`で上書き可能）。

クイックオープン（ローカルGateway）:

* [http://127.0.0.1:18789/](http://127.0.0.1:18789/)（または [http://localhost:18789/](http://localhost:18789/)）

主な参照:

* [Control UI](/web/control-ui) 使用法とUI機能について。
* [Tailscale](/gateway/tailscale) Serve/Funnelの自動化について。
* [Web surfaces](/web) バインドモードとセキュリティノートについて。

認証は`connect.params.auth`を介してWebSocketハンドシェイクで強制されます
（トークンまたはパスワード）。`gateway.auth`については[Gateway configuration](/gateway/configuration)を参照してください。

セキュリティノート: Control UIは**管理者用インターフェース**です（チャット、設定、実行の承認）。
公開しないでください。UIは初回読み込み後にトークンを`localStorage`に保存します。
localhost、Tailscale Serve、またはSSHトンネルを使用することをお勧めします。

## ファストパス（推奨）

* オンボーディング後、CLIはダッシュボードを自動的に開き、クリーンな（トークンなしの）リンクを表示します。
* いつでも再オープン: `openclaw dashboard`（リンクをコピーし、可能であればブラウザを開き、ヘッドレスの場合はSSHヒントを表示します）。
* UIが認証を要求する場合は、`gateway.auth.token`（または`OPENCLAW_GATEWAY_TOKEN`）からトークンをControl UI設定に貼り付けて接続します。

## トークンの基本（ローカル vs リモート）

* **ローカルホスト**: `http://127.0.0.1:18789/`を開く。
* **トークンソース**: `gateway.auth.token`（または`OPENCLAW_GATEWAY_TOKEN`）；接続後、UIはlocalStorageにコピーを保存します。
* **ローカルホストでない場合**: Tailscale Serveを使用（`gateway.auth.allowTailscale: true`の場合、Control UI/WebSocketはトークンなしで使用可能、信頼されたGatewayホストを前提とします；HTTP APIは依然としてトークン/パスワードが必要）、トークンを使用したtailnetバインド、またはSSHトンネルを使用します。詳細は[Web surfaces](/web)を参照してください。

## “unauthorized” / 1008が表示された場合

* Gatewayにアクセスできることを確認してください（ローカル: `openclaw status`；リモート: SSHトンネル `ssh -N -L 18789:127.0.0.1:18789 user@host` その後 `http://127.0.0.1:18789/`を開く）。
* Gatewayホストからトークンを取得します: `openclaw config get gateway.auth.token`（または生成します: `openclaw doctor --generate-gateway-token`）。
* ダッシュボード設定で、トークンを認証フィールドに貼り付けて接続します。
