# Talk モード

# Talk モード

Talkモードは、継続的な音声会話ループです：

1. 音声を聞く
2. トランスクリプトをモデルに送信する（メインセッション、chat.send）
3. 応答を待つ
4. ElevenLabsを介して話す（ストリーミング再生）

## 動作（macOS）

* Talkモードが有効な間は**常時表示のオーバーレイ**。
* **聴取 → 思考 → 話す**のフェーズ遷移。
* **短いポーズ**（サイレンスウィンドウ）では、現在のトランスクリプトが送信されます。
* 返信は**WebChatに書き込まれる**（タイピングと同じ）。
* **音声による中断**（デフォルトオン）：ユーザーがアシスタントが話している間に話し始めると、再生を停止し、次のプロンプトのために中断のタイムスタンプを記録します。

## 返信における音声指示

アシスタントは、音声を制御するために**単一のJSON行**で返信を接頭辞することがあります：

```json
{ "voice": "<voice-id>", "once": true }
```

ルール：

* 最初の非空行のみ。
* 不明なキーは無視されます。
* `once: true`は現在の返信のみに適用されます。
* `once`がない場合、音声はTalkモードの新しいデフォルトになります。
* JSON行はTTS再生前に削除されます。

サポートされているキー：

* `voice` / `voice_id` / `voiceId`
* `model` / `model_id` / `modelId`
* `speed`, `rate` (WPM), `stability`, `similarity`, `style`, `speakerBoost`
* `seed`, `normalize`, `lang`, `output_format`, `latency_tier`
* `once`

## 設定（`~/.openclaw/openclaw.json`）

```json5
{
  talk: {
    voiceId: "elevenlabs_voice_id",
    modelId: "eleven_v3",
    outputFormat: "mp3_44100_128",
    apiKey: "elevenlabs_api_key",
    interruptOnSpeech: true,
  },
}
```

デフォルト：

* `interruptOnSpeech`: true
* `voiceId`: `ELEVENLABS_VOICE_ID` / `SAG_VOICE_ID`（またはAPIキーが利用可能な場合は最初のElevenLabs音声）にフォールバック
* `modelId`: 設定されていない場合は`eleven_v3`がデフォルト
* `apiKey`: `ELEVENLABS_API_KEY`（または利用可能な場合はゲートウェイシェルプロファイル）にフォールバック
* `outputFormat`: macOS/iOSでは`pcm_44100`、Androidでは`pcm_24000`がデフォルト（MP3ストリーミングを強制するには`mp3_*`を設定）

## macOS UI

* メニューバーのトグル：**Talk**
* 設定タブ：**Talk Mode**グループ（音声ID + 中断トグル）
* オーバーレイ：
  * **Listening**：マイクレベルでパルスするクラウド
  * **Thinking**：沈むアニメーション
  * **Speaking**：放射状のリング
  * クラウドをクリック：話すのを停止
  * Xをクリック：Talkモードを終了

## 注意事項

* 音声 + マイクの権限が必要です。
* セッションキー`main`に対して`chat.send`を使用します。
* TTSは、低遅延のためにmacOS/iOS/Androidで`ELEVENLABS_API_KEY`を使用したElevenLabsストリーミングAPIとインクリメンタル再生を使用します。
* `eleven_v3`の`stability`は`0.0`、`0.5`、または`1.0`に検証されています。他のモデルは`0..1`を受け入れます。
* `latency_tier`は設定時に`0..4`に検証されます。
* Androidは、低遅延のAudioTrackストリーミングのために`pcm_16000`、`pcm_22050`、`pcm_24000`、および`pcm_44100`出力フォーマットをサポートしています。
