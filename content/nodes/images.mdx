# 画像とメディアサポート

# Image & Media Support — 2025-12-05

WhatsAppチャネルは**Baileys Web**を介して動作します。このドキュメントは、送信、ゲートウェイ、およびエージェントの応答に関する現在のメディア処理ルールをまとめたものです。

## 目標

* `openclaw message send --media`を使用して、オプションのキャプション付きでメディアを送信する。
* ウェブ受信トレイからの自動応答にテキストと一緒にメディアを含めることを許可する。
* タイプごとの制限を健全で予測可能なものに保つ。

## CLIインターフェース

* `openclaw message send --media <path-or-url> [--message <caption>]`
  * `--media`はオプションであり、キャプションはメディアのみの送信の場合は空にできます。
  * `--dry-run`は解決されたペイロードを表示します; `--json`は`{ channel, to, messageId, mediaUrl, caption }`を出力します。

## WhatsApp Webチャネルの動作

* 入力: ローカルファイルパス**または**HTTP(S) URL。
* フロー: バッファにロードし、メディアの種類を検出し、正しいペイロードを構築します:
  * **画像:** JPEGにリサイズおよび再圧縮（最大サイド2048px）、`agents.defaults.mediaMaxMb`（デフォルト5MB）をターゲットにし、6MBに制限。
  * **音声/ボイス/ビデオ:** 最大16MBまでパススルー; 音声はボイスノートとして送信されます（`ptt: true`）。
  * **ドキュメント:** その他すべて、最大100MBで、利用可能な場合はファイル名を保持します。
* WhatsApp GIFスタイルの再生: `gifPlayback: true`を指定してMP4を送信します（CLI: `--gif-playback`）ので、モバイルクライアントはインラインでループします。
* MIME検出はマジックバイトを優先し、その後ヘッダー、ファイル拡張子の順です。
* キャプションは`--message`または`reply.text`から取得され、空のキャプションも許可されます。
* ロギング: 非冗長表示は`↩️`/`✅`を示します; 冗長表示にはサイズとソースパス/URLが含まれます。

## 自動応答パイプライン

* `getReplyFromConfig`は`{ text?, mediaUrl?, mediaUrls? }`を返します。
* メディアが存在する場合、ウェブ送信者は`openclaw message send`と同じパイプラインを使用してローカルパスまたはURLを解決します。
* 複数のメディアエントリは、提供された場合、順次送信されます。

## コマンドへの受信メディア (Pi)

* 受信ウェブメッセージにメディアが含まれている場合、OpenClawは一時ファイルにダウンロードし、テンプレート変数を公開します:
  * `{{MediaUrl}}`は受信メディアの擬似URLです。
  * `{{MediaPath}}`はコマンドを実行する前に書き込まれたローカルの一時パスです。
* セッションごとのDockerサンドボックスが有効な場合、受信メディアはサンドボックスのワークスペースにコピーされ、`MediaPath`/`MediaUrl`は`media/inbound/<filename>`のような相対パスに書き換えられます。
* メディア理解（`tools.media.*`または共有`tools.media.models`を介して構成された場合）は、テンプレート処理の前に実行され、`Body`に`[Image]`、`[Audio]`、および`[Video]`ブロックを挿入できます。
  * 音声は`{{Transcript}}`を設定し、コマンド解析のためにトランスクリプトを使用するため、スラッシュコマンドは引き続き機能します。
  * ビデオおよび画像の説明は、コマンド解析のためにキャプションテキストを保持します。
* デフォルトでは、最初の一致する画像/音声/ビデオ添付ファイルのみが処理されます; 複数の添付ファイルを処理するには`tools.media.<cap>.attachments`を設定します。

## 制限とエラー

**送信制限 (WhatsAppウェブ送信)**

* 画像: 再圧縮後の約6MB制限。
* 音声/ボイス/ビデオ: 16MB制限; ドキュメント: 100MB制限。
* サイズオーバーまたは読み取れないメディア → ログに明確なエラーが表示され、応答はスキップされます。

**メディア理解の制限 (文字起こし/説明)**

* 画像デフォルト: 10MB (`tools.media.image.maxBytes`)。
* 音声デフォルト: 20MB (`tools.media.audio.maxBytes`)。
* ビデオデフォルト: 50MB (`tools.media.video.maxBytes`)。
* サイズオーバーのメディアは理解をスキップしますが、応答は元の本文で通過します。

## テストのための注意事項

* 画像/音声/ドキュメントのケースに対して送信+応答フローをカバーします。
* 画像の再圧縮（サイズ制限）と音声のボイスノートフラグを検証します。
* マルチメディアの応答が順次送信としてファンアウトすることを確認します。
