# 位置情報コマンド

# Location command (nodes)

## TL;DR

* `location.get` はノードコマンドです（`node.invoke`経由）。
* デフォルトではオフ。
* 設定はセレクタを使用します: オフ / 使用中 / 常に。
* 別のトグル: 精密位置情報。

## なぜセレクタが必要なのか（単なるスイッチではない理由）

OSの権限は多層的です。アプリ内でセレクタを表示することはできますが、実際の許可はOSが決定します。

* iOS/macOS: ユーザーはシステムのプロンプト/設定で**使用中**または**常に**を選択できます。アプリはアップグレードをリクエストできますが、OSは設定を必要とする場合があります。
* Android: バックグラウンド位置情報は別の権限です。Android 10以上では、設定フローが必要なことが多いです。
* 精密位置情報は別の許可です（iOS 14以上の「精密」、Androidの「fine」対「coarse」）。

UIのセレクタは、私たちが要求するモードを決定します。実際の許可はOSの設定にあります。

## 設定モデル

ノードデバイスごとに:

* `location.enabledMode`: `off | whileUsing | always`
* `location.preciseEnabled`: bool

UIの動作:

* `whileUsing`を選択すると、フォアグラウンドの権限を要求します。
* `always`を選択すると、まず`whileUsing`が確保され、その後バックグラウンドを要求します（必要な場合はユーザーを設定に送ります）。
* OSが要求されたレベルを拒否した場合、最も高い許可されたレベルに戻り、ステータスを表示します。

## 権限マッピング (node.permissions)

オプションです。macOSノードは権限マップを介して`location`を報告しますが、iOS/Androidでは省略されることがあります。

## コマンド: `location.get`

`node.invoke`経由で呼び出されます。

パラメータ（推奨）:

```json
{
  "timeoutMs": 10000,
  "maxAgeMs": 15000,
  "desiredAccuracy": "coarse|balanced|precise"
}
```

レスポンスペイロード:

```json
{
  "lat": 48.20849,
  "lon": 16.37208,
  "accuracyMeters": 12.5,
  "altitudeMeters": 182.0,
  "speedMps": 0.0,
  "headingDeg": 270.0,
  "timestamp": "2026-01-03T12:34:56.000Z",
  "isPrecise": true,
  "source": "gps|wifi|cell|unknown"
}
```

エラー（安定したコード）:

* `LOCATION_DISABLED`: セレクタがオフです。
* `LOCATION_PERMISSION_REQUIRED`: 要求されたモードの権限が不足しています。
* `LOCATION_BACKGROUND_UNAVAILABLE`: アプリはバックグラウンドですが、使用中のみが許可されています。
* `LOCATION_TIMEOUT`: 時間内に位置情報を取得できませんでした。
* `LOCATION_UNAVAILABLE`: システムの障害 / プロバイダーがありません。

## バックグラウンド動作（将来）

目標: モデルはノードがバックグラウンドにあるときでも位置情報を要求できるが、次の条件を満たす場合のみ:

* ユーザーが**常に**を選択した。
* OSがバックグラウンド位置情報を許可した。
* アプリが位置情報のためにバックグラウンドで実行されることが許可されている（iOSバックグラウンドモード / Androidフォアグラウンドサービスまたは特別な許可）。

プッシュトリガーされたフロー（将来）:

1. Gatewayがノードにプッシュを送信します（サイレントプッシュまたはFCMデータ）。
2. ノードが短時間ウェイクアップし、デバイスから位置情報を要求します。
3. ノードがペイロードをGatewayに転送します。

注意:

* iOS: 常にの権限 + バックグラウンド位置情報モードが必要です。サイレントプッシュは制限される場合があります; 不定期な失敗が予想されます。
* Android: バックグラウンド位置情報はフォアグラウンドサービスを必要とする場合があります; そうでなければ、拒否されることが予想されます。

## モデル/ツール統合

* ツールサーフェス: `nodes`ツールは`location_get`アクションを追加します（ノードが必要）。
* CLI: `openclaw nodes location get --node <id>`。
* エージェントガイドライン: ユーザーが位置情報を有効にし、範囲を理解している場合のみ呼び出します。

## UXコピー（推奨）

* オフ: “位置情報共有は無効です。”
* 使用中: “OpenClawが開いているときのみ。”
* 常に: “バックグラウンド位置情報を許可します。システムの権限が必要です。”
* 精密: “精密GPS位置情報を使用します。おおよその位置情報を共有するにはオフにしてください。”
