# Voice Wake

# Voice Wake (グローバルウェイクワード)

OpenClawは**ウェイクワードを単一のグローバルリスト**として扱い、これは**Gateway**が所有しています。

* **ノードごとのカスタムウェイクワードはありません**。
* **任意のノード/アプリのUIが**リストを編集できます。変更はGatewayによって永続化され、全員にブロードキャストされます。
* 各デバイスは依然として独自の**Voice Wake 有効/無効**トグルを保持しています（ローカルUX + 権限は異なります）。

## ストレージ (Gatewayホスト)

ウェイクワードはゲートウェイマシンの以下の場所に保存されます：

* `~/.openclaw/settings/voicewake.json`

形状：

```json
{ "triggers": ["openclaw", "claude", "computer"], "updatedAtMs": 1730000000000 }
```

## プロトコル

### メソッド

* `voicewake.get` → `{ triggers: string[] }`
* `voicewake.set` with params `{ triggers: string[] }` → `{ triggers: string[] }`

注意事項：

* トリガーは正規化されます（トリム、空のものは削除）。空のリストはデフォルトに戻ります。
* 安全のために制限が適用されます（カウント/長さの上限）。

### イベント

* `voicewake.changed` ペイロード `{ triggers: string[] }`

受信者：

* すべてのWebSocketクライアント（macOSアプリ、WebChatなど）
* すべての接続されたノード（iOS/Android）、およびノード接続時に初期の「現在の状態」プッシュとしても。

## クライアントの動作

### macOSアプリ

* グローバルリストを使用して`VoiceWakeRuntime`トリガーを制御します。
* Voice Wake設定で「トリガーワード」を編集すると`voicewake.set`が呼び出され、その後ブロードキャストに依存して他のクライアントを同期させます。

### iOSノード

* `VoiceWakeManager`トリガー検出のためにグローバルリストを使用します。
* 設定でウェイクワードを編集すると`voicewake.set`が呼び出され（Gateway WS経由）、ローカルのウェイクワード検出も応答性を保ちます。

### Androidノード

* 設定にウェイクワードエディタを表示します。
* `voicewake.set`をGateway WS経由で呼び出し、編集内容がどこでも同期されるようにします。
