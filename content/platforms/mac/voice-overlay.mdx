# Voice Overlay

# Voice Overlay ライフサイクル (macOS)

対象: macOS アプリの貢献者。目標: ウェイクワードとプッシュトークが重なったときにボイスオーバーレイを予測可能に保つ。

## 現在の意図

* オーバーレイがウェイクワードから既に表示されている場合にユーザーがホットキーを押すと、ホットキーセッションは既存のテキストをリセットするのではなく*採用*します。ホットキーが押されている間、オーバーレイは表示されたままになります。ユーザーがリリースすると、トリムされたテキストがあれば送信し、なければ解消します。
* ウェイクワード単独では、サイレンス時に自動送信されます。プッシュトークはリリース時に即座に送信されます。

## 実装済み (2025年12月9日)

* オーバーレイセッションは、キャプチャごとにトークンを持つようになりました（ウェイクワードまたはプッシュトーク）。トークンが一致しない場合、部分的/最終的/送信/解消/レベルの更新は破棄され、古いコールバックを避けます。
* プッシュトークは、表示されているオーバーレイテキストをプレフィックスとして採用します（ウェイクオーバーレイが表示されている間にホットキーを押すと、テキストが保持され、新しいスピーチが追加されます）。最終的なトランスクリプトを得るために最大1.5秒待機し、その後現在のテキストにフォールバックします。
* チャイム/オーバーレイのログは、`info`レベルで`voicewake.overlay`、`voicewake.ptt`、および`voicewake.chime`のカテゴリに出力されます（セッション開始、部分的、最終的、送信、解消、チャイムの理由）。

## 次のステップ

1. **VoiceSessionCoordinator (アクター)**
   * 一度に正確に1つの`VoiceSession`を所有します。
   * API（トークンベース）: `beginWakeCapture`、`beginPushToTalk`、`updatePartial`、`endCapture`、`cancel`、`applyCooldown`。
   * 古いトークンを持つコールバックを破棄します（古い認識器がオーバーレイを再オープンするのを防ぎます）。
2. **VoiceSession (モデル)**
   * フィールド: `token`、`source`（wakeWord|pushToTalk）、コミットされた/揮発性のテキスト、チャイムフラグ、タイマー（自動送信、アイドル）、`overlayMode`（表示|編集|送信）、クールダウンの期限。
3. **オーバーレイバインディング**
   * `VoiceSessionPublisher`（`ObservableObject`）は、アクティブなセッションをSwiftUIにミラーリングします。
   * `VoiceWakeOverlayView`は、パブリッシャーを介してのみレンダリングされ、グローバルなシングルトンを直接変更することはありません。
   * オーバーレイのユーザーアクション（`sendNow`、`dismiss`、`edit`）は、セッショントークンを持ってコーディネーターにコールバックします。
4. **統一された送信パス**
   * `endCapture`で: トリムされたテキストが空の場合 → 解消; そうでなければ`performSend(session:)`（送信チャイムを1回鳴らし、前進し、解消します）。
   * プッシュトーク: 遅延なし; ウェイクワード: 自動送信のためのオプションの遅延。
   * プッシュトークが終了した後、ウェイクランタイムに短いクールダウンを適用し、ウェイクワードが即座に再トリガーされないようにします。
5. **ログ記録**
   * コーディネーターは、サブシステム`ai.openclaw`、カテゴリ`voicewake.overlay`および`voicewake.chime`で`.info`ログを出力します。
   * 主要なイベント: `session_started`、`adopted_by_push_to_talk`、`partial`、`finalized`、`send`、`dismiss`、`cancel`、`cooldown`。

## デバッグチェックリスト

* スティッキーオーバーレイを再現しながらログをストリームします:

  ```bash
  sudo log stream --predicate 'subsystem == "ai.openclaw" AND category CONTAINS "voicewake"' --level info --style compact
  ```

* アクティブなセッショントークンが1つだけであることを確認します。古いコールバックはコーディネーターによって破棄されるべきです。

* プッシュトークのリリースが常にアクティブなトークンで`endCapture`を呼び出すことを確認します。テキストが空の場合、チャイムや送信なしで`dismiss`を期待します。

## マイグレーションステップ（提案）

1. `VoiceSessionCoordinator`、`VoiceSession`、および`VoiceSessionPublisher`を追加します。
2. `VoiceWakeRuntime`をリファクタリングして、`VoiceWakeOverlayController`に直接触れるのではなく、セッションを作成/更新/終了します。
3. `VoicePushToTalk`をリファクタリングして、既存のセッションを採用し、リリース時に`endCapture`を呼び出します。ランタイムのクールダウンを適用します。
4. `VoiceWakeOverlayController`をパブリッシャーに接続し、ランタイム/PTTからの直接呼び出しを削除します。
5. セッションの採用、クールダウン、および空テキストの解消に対する統合テストを追加します。
