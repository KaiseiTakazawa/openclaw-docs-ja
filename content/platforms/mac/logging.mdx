# macOS ロギング

# Logging (macOS)

## ローリング診断ファイルログ (デバッグペイン)

OpenClawはmacOSアプリのログをswift-logを通じてルーティングし（デフォルトで統一ログ）、耐久性のあるキャプチャが必要な場合にはローカルの回転ファイルログをディスクに書き込むことができます。

* 詳細度: **デバッグペイン → ログ → アプリログ → 詳細度**
* 有効化: **デバッグペイン → ログ → アプリログ → “ローリング診断ログを書き込む (JSONL)”**
* 位置: `~/Library/Logs/OpenClaw/diagnostics.jsonl` (自動的に回転; 古いファイルは`.1`, `.2`, …の接尾辞が付く)
* 消去: **デバッグペイン → ログ → アプリログ → “消去”**

注意事項:

* これは**デフォルトでオフ**です。アクティブにデバッグしている間のみ有効にしてください。
* このファイルは機密情報として扱ってください; レビューなしで共有しないでください。

## macOSにおける統一ログのプライベートデータ

統一ログは、サブシステムが`privacy -off`にオプトインしない限り、ほとんどのペイロードを隠します。PeterのmacOSに関する[ログプライバシーのいたずら](https://steipete.me/posts/2025/logging-privacy-shenanigans)（2025年）によれば、これは`/Library/Preferences/Logging/Subsystems/`内のplistによって制御され、サブシステム名でキー付けされています。新しいログエントリのみがフラグを取得するため、問題を再現する前に有効にしてください。

## OpenClawの有効化 (`ai.openclaw`)

* まずplistを一時ファイルに書き込み、その後ルートとして原子的にインストールします:

```bash
cat <<'EOF' >/tmp/ai.openclaw.plist
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>DEFAULT-OPTIONS</key>
    <dict>
        <key>Enable-Private-Data</key>
        <true/>
    </dict>
</dict>
</plist>
EOF
sudo install -m 644 -o root -g wheel /tmp/ai.openclaw.plist /Library/Preferences/Logging/Subsystems/ai.openclaw.plist
```

* 再起動は必要ありません; logdはファイルをすぐに認識しますが、新しいログ行のみがプライベートペイロードを含みます。
* 既存のヘルパーを使用して、より豊富な出力を表示します。例: `./scripts/clawlog.sh --category WebChat --last 5m`。

## デバッグ後の無効化

* オーバーライドを削除します: `sudo rm /Library/Preferences/Logging/Subsystems/ai.openclaw.plist`。
* オプションで`sudo log config --reload`を実行して、logdにオーバーライドを即座に解除させます。
* この情報には電話番号やメッセージ本文が含まれる可能性があることを忘れないでください; 追加の詳細が必要な間だけplistをそのままにしておいてください。
