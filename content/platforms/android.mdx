# Android アプリ

# Android App (Node)

## Support snapshot

* 役割: コンパニオンノードアプリ (Android は Gateway をホストしません)。
* Gateway 必要: はい (macOS、Linux、または Windows 上で WSL2 経由で実行)。
* インストール: [Getting Started](/start/getting-started) + [Pairing](/gateway/pairing)。
* Gateway: [Runbook](/gateway) + [Configuration](/gateway/configuration)。
  * プロトコル: [Gateway protocol](/gateway/protocol) (ノード + コントロールプレーン)。

## System control

システムコントロール (launchd/systemd) は Gateway ホスト上に存在します。詳細は [Gateway](/gateway) を参照してください。

## Connection Runbook

Android ノードアプリ ⇄ (mDNS/NSD + WebSocket) ⇄ **Gateway**

Android は直接 Gateway WebSocket に接続します (デフォルト `ws://<host>:18789`) そして Gateway 所有のペアリングを使用します。

### 前提条件

* “マスター” マシン上で Gateway を実行できます。
* Android デバイス/エミュレーターが Gateway WebSocket に到達できる:
  * mDNS/NSD を使用した同じ LAN、**または**
  * Wide-Area Bonjour / ユニキャスト DNS-SD を使用した同じ Tailscale テイルネット (下記参照)、**または**
  * 手動 Gateway ホスト/ポート (フォールバック)
* Gateway マシン上で CLI (`openclaw`) を実行できます (または SSH 経由で)。

### 1) Gateway を起動する

```bash
openclaw gateway --port 18789 --verbose
```

ログに以下のようなメッセージが表示されることを確認してください:

* `listening on ws://0.0.0.0:18789`

テイルネット専用のセットアップ (ウィーン ⇄ ロンドンに推奨) の場合、Gateway をテイルネット IP にバインドします:

* Gateway ホストの `~/.openclaw/openclaw.json` に `gateway.bind: "tailnet"` を設定します。
* Gateway / macOS メニューバーアプリを再起動します。

### 2) 発見を確認する (オプション)

Gateway マシンから:

```bash
dns-sd -B _openclaw-gw._tcp local.
```

さらなるデバッグノート: [Bonjour](/gateway/bonjour)。

#### テイルネット (ウィーン ⇄ ロンドン) のユニキャスト DNS-SD を介した発見

Android の NSD/mDNS 発見はネットワークを越えません。Android ノードと Gateway が異なるネットワーク上にあるが Tailscale 経由で接続されている場合は、Wide-Area Bonjour / ユニキャスト DNS-SD を使用してください:

1. Gateway ホスト上で DNS-SD ゾーン (例: `openclaw.internal.`) を設定し、`_openclaw-gw._tcp` レコードを公開します。
2. 選択したドメインに対してその DNS サーバーを指す Tailscale スプリット DNS を構成します。

詳細と例の CoreDNS 設定: [Bonjour](/gateway/bonjour)。

### 3) Android から接続する

Android アプリ内で:

* アプリは **フォアグラウンドサービス** (永続通知) を介して Gateway 接続を維持します。
* **設定**を開きます。
* **発見された Gateway** の下で、あなたの Gateway を選択し、**接続**をクリックします。
* mDNS がブロックされている場合は、**詳細 → 手動 Gateway** (ホスト + ポート) を使用し、**接続 (手動)** をクリックします。

最初のペアリングが成功した後、Android は起動時に自動的に再接続します:

* 手動エンドポイント (有効な場合)、さもなくば
* 最後に発見された Gateway (ベストエフォート)。

### 4) ペアリングを承認する (CLI)

Gateway マシン上で:

```bash
openclaw nodes pending
openclaw nodes approve <requestId>
```

ペアリングの詳細: [Gateway pairing](/gateway/pairing)。

### 5) ノードが接続されていることを確認する

* ノードのステータスを介して:

  ```bash
  openclaw nodes status
  ```

* Gateway を介して:

  ```bash
  openclaw gateway call node.list --params "{}"
  ```

### 6) チャット + 履歴

Android ノードのチャットシートは Gateway の **プライマリセッションキー** (`main`) を使用するため、履歴と返信は WebChat や他のクライアントと共有されます:

* 履歴: `chat.history`
* 送信: `chat.send`
* プッシュ更新 (ベストエフォート): `chat.subscribe` → `event:"chat"`

### 7) Canvas + カメラ

#### Gateway Canvas ホスト (ウェブコンテンツに推奨)

ノードがエージェントがディスク上で編集できる実際の HTML/CSS/JS を表示する場合、ノードを Gateway canvas ホストにポイントします。

注意: ノードは Gateway HTTP サーバーからキャンバスを読み込みます (同じポートは `gateway.port`、デフォルトは `18789`)。

1. Gateway ホスト上に `~/.openclaw/workspace/canvas/index.html` を作成します。

2. ノードをそれにナビゲートします (LAN):

```bash
openclaw nodes invoke --node "<Android Node>" --command canvas.navigate --params '{"url":"http://<gateway-hostname>.local:18789/__openclaw__/canvas/"}'
```

テイルネット (オプション): 両方のデバイスが Tailscale 上にある場合、`.local` の代わりに MagicDNS 名または テイルネット IP を使用します。例: `http://<gateway-magicdns>:18789/__openclaw__/canvas/`。

このサーバーは HTML にライブリロードクライアントを注入し、ファイル変更時にリロードします。
A2UI ホストは `http://<gateway-host>:18789/__openclaw__/a2ui/` にあります。

Canvas コマンド (フォアグラウンドのみ):

* `canvas.eval`, `canvas.snapshot`, `canvas.navigate` (デフォルトのスカフォールドに戻るには `{"url":""}` または `{"url":"/"}` を使用)。 `canvas.snapshot` は `{ format, base64 }` を返します (デフォルト `format="jpeg"`).
* A2UI: `canvas.a2ui.push`, `canvas.a2ui.reset` (`canvas.a2ui.pushJSONL` のレガシーエイリアス)

カメラコマンド (フォアグラウンドのみ; 権限が必要):

* `camera.snap` (jpg)
* `camera.clip` (mp4)

パラメータと CLI ヘルパーについては [Camera node](/nodes/camera) を参照してください。
