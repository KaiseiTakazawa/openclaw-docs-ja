# ACP エージェント

# ACP agents

[Agent Client Protocol (ACP)](https://agentclientprotocol.com/) セッションは、OpenClaw が外部コーディングハーネス（例えば Pi、Claude Code、Codex、OpenCode、Gemini CLI）を ACP バックエンドプラグインを通じて実行できるようにします。

OpenClaw に「これを Codex で実行して」または「スレッドで Claude Code を開始して」といった平易な言葉で尋ねると、OpenClaw はそのリクエストを ACP ランタイム（ネイティブサブエージェントランタイムではなく）にルーティングするべきです。

## 高速オペレーターフロー

実用的な `/acp` ランブックが必要なときに使用します：

1. セッションを生成する：
   * `/acp spawn codex --mode persistent --thread auto`
2. バウンドスレッドで作業する（またはそのセッションキーを明示的にターゲットにする）。
3. ランタイム状態を確認する：
   * `/acp status`
4. 必要に応じてランタイムオプションを調整する：
   * `/acp model <provider/model>`
   * `/acp permissions <profile>`
   * `/acp timeout <seconds>`
5. コンテキストを置き換えずにアクティブなセッションを調整する：
   * `/acp steer tighten logging and continue`
6. 作業を停止する：
   * `/acp cancel`（現在のターンを停止）、または
   * `/acp close`（セッションを閉じてバインディングを削除）

## 人間向けのクイックスタート

自然なリクエストの例：

* 「ここでスレッド内に持続的な Codex セッションを開始し、集中させておいてください。」
* 「これを一度きりの Claude Code ACP セッションとして実行し、結果を要約してください。」
* 「このタスクに Gemini CLI をスレッドで使用し、その後のフォローアップを同じスレッドで続けてください。」

OpenClaw が行うべきこと：

1. `runtime: "acp"` を選択する。
2. リクエストされたハーネスターゲット（例えば `agentId` として `codex`）を解決する。
3. スレッドバインディングが要求され、現在のチャネルがそれをサポートしている場合、ACP セッションをスレッドにバインドする。
4. フォローアップスレッドメッセージをその同じ ACP セッションにルーティングし、焦点を外す/閉じる/期限切れになるまで続ける。

## ACP とサブエージェントの違い

外部ハーネスランタイムが必要な場合は ACP を使用します。OpenClaw ネイティブの委任実行が必要な場合はサブエージェントを使用します。

| エリア          | ACP セッション                           | サブエージェント実行                      |
| ------------- | ------------------------------------- | ---------------------------------- |
| ランタイム       | ACP バックエンドプラグイン（例えば acpx） | OpenClaw ネイティブサブエージェントランタイム  |
| セッションキー   | `agent:<agentId>:acp:<uuid>`          | `agent:<agentId>:subagent:<uuid>`  |
| 主なコマンド | `/acp ...`                            | `/subagents ...`                   |
| スポーンツール    | `sessions_spawn` で `runtime:"acp"` | `sessions_spawn`（デフォルトランタイム） |

さらに詳しくは [Sub-agents](/tools/subagents) を参照してください。

## スレッドバウンドセッション（チャネル非依存）

チャネルアダプタに対してスレッドバインディングが有効な場合、ACP セッションはスレッドにバインドできます：

* OpenClaw はスレッドをターゲット ACP セッションにバインドします。
* そのスレッド内のフォローアップメッセージはバインドされた ACP セッションにルーティングされます。
* ACP の出力は同じスレッドに返されます。
* 焦点を外す/閉じる/アーカイブ/アイドルタイムアウトまたは最大年齢の期限切れがバインディングを削除します。

スレッドバインディングのサポートはアダプタ固有です。アクティブなチャネルアダプタがスレッドバインディングをサポートしていない場合、OpenClaw は明確なサポートされていない/利用できないメッセージを返します。

スレッドバウンド ACP に必要な機能フラグ：

* `acp.enabled=true`
* `acp.dispatch.enabled=true`
* チャネルアダプタ ACP スレッドスポーンフラグが有効（アダプタ固有）
  * Discord: `channels.discord.threadBindings.spawnAcpSessions=true`

### スレッドをサポートするチャネル

* セッション/スレッドバインディング機能を公開する任意のチャネルアダプタ。
* 現在のビルトインサポート: Discord。
* プラグインチャネルは同じバインディングインターフェースを通じてサポートを追加できます。

## ACP セッションの開始（インターフェース）

### `sessions_spawn` から

エージェントターンまたはツールコールから ACP セッションを開始するには `runtime: "acp"` を使用します。

```json
{
  "task": "リポジトリを開いて失敗したテストを要約する",
  "runtime": "acp",
  "agentId": "codex",
  "thread": true,
  "mode": "session"
}
```

注意：

* `runtime` はデフォルトで `subagent` となるため、ACP セッションのために `runtime: "acp"` を明示的に設定してください。
* `agentId` が省略されると、OpenClaw は設定されている場合 `acp.defaultAgent` を使用します。
* `mode: "session"` は持続的なバウンド会話を維持するために `thread: true` を必要とします。

インターフェースの詳細：

* `task`（必須）：ACP セッションに送信される初期プロンプト。
* `runtime`（ACP に必須）：`"acp"` でなければなりません。
* `agentId`（オプション）：ACP ターゲットハーネス ID。設定されている場合は `acp.defaultAgent` にフォールバックします。
* `thread`（オプション、デフォルト `false`）：サポートされている場合にスレッドバインディングフローを要求します。
* `mode`（オプション）：`run`（一度きり）または `session`（持続的）。
  * デフォルトは `run`
  * `thread: true` でモードが省略されると、OpenClaw はランタイムパスごとに持続的な動作をデフォルトにする場合があります
  * `mode: "session"` は `thread: true` を必要とします。
* `cwd`（オプション）：要求されたランタイム作業ディレクトリ（バックエンド/ランタイムポリシーによって検証されます）。
* `label`（オプション）：セッション/バナーのテキストに使用されるオペレーター向けのラベル。

### `/acp` コマンドから

必要に応じてチャットから明示的なオペレーターコントロールのために `/acp spawn` を使用します。

```text
/acp spawn codex --mode persistent --thread auto
/acp spawn codex --mode oneshot --thread off
/acp spawn codex --thread here
```

主要なフラグ：

* `--mode persistent|oneshot`
* `--thread auto|here|off`
* `--cwd <absolute-path>`
* `--label <name>`

[Slash Commands](/tools/slash-commands) を参照してください。

## セッションターゲット解決

ほとんどの `/acp` アクションはオプションのセッションターゲット（`session-key`、`session-id`、または `session-label`）を受け入れます。

解決順序：

1. 明示的なターゲット引数（または `/acp steer` のための `--session`）
   * キーを試す
   * 次に UUID 形式のセッション ID
   * 次にラベル
2. 現在のスレッドバインディング（この会話/スレッドが ACP セッションにバインドされている場合）
3. 現在のリクエスターセッションのフォールバック

ターゲットが解決できない場合、OpenClaw は明確なエラーを返します（`Unable to resolve session target: ...`）。

## スポーンスレッドモード

`/acp spawn` は `--thread auto|here|off` をサポートします。

| モード   | 挙動                                                                                            |
| ------ | --------------------------------------------------------------------------------------------------- |
| `auto` | アクティブなスレッド内：そのスレッドをバインドします。スレッド外：サポートされている場合に子スレッドを作成/バインドします。 |
| `here` | 現在のアクティブスレッドを要求し、そうでない場合は失敗します。                                                  |
| `off`  | バインディングなし。セッションは未バインドで開始します。                                                                 |

注意：

* 非スレッドバインディングの表面では、デフォルトの挙動は実質的に `off` です。
* スレッドバウンドスポーンはチャネルポリシーサポートを必要とします（Discord の場合：`channels.discord.threadBindings.spawnAcpSessions=true`）。

## ACP コントロール

利用可能なコマンドファミリー：

* `/acp spawn`
* `/acp cancel`
* `/acp steer`
* `/acp close`
* `/acp status`
* `/acp set-mode`
* `/acp set`
* `/acp cwd`
* `/acp permissions`
* `/acp timeout`
* `/acp model`
* `/acp reset-options`
* `/acp sessions`
* `/acp doctor`
* `/acp install`

`/acp status` は有効なランタイムオプションを表示し、利用可能な場合はランタイムレベルおよびバックエンドレベルのセッション識別子の両方を表示します。

一部のコントロールはバックエンドの機能に依存します。バックエンドがコントロールをサポートしていない場合、OpenClaw は明確なサポートされていないコントロールエラーを返します。

## ACP コマンドクックブック

| コマンド              | 何をするか                                              | 例                                                              |
| -------------------- | --------------------------------------------------------- | -------------------------------------------------------------- |
| `/acp spawn`         | ACP セッションを作成します。オプションでスレッドをバインドします。                 | `/acp spawn codex --mode persistent --thread auto --cwd /repo` |
| `/acp cancel`        | ターゲットセッションの進行中のターンをキャンセルします。                 | `/acp cancel agent:codex:acp:<uuid>`                           |
| `/acp steer`         | 実行中のセッションにステア指示を送信します。                | `/acp steer --session support inbox prioritize failing tests`  |
| `/acp close`         | セッションを閉じてスレッドターゲットのバインディングを解除します。                  | `/acp close`                                                   |
| `/acp status`        | バックエンド、モード、状態、ランタイムオプション、機能を表示します。 | `/acp status`                                                  |
| `/acp set-mode`      | ターゲットセッションのランタイムモードを設定します。                      | `/acp set-mode plan`                                           |
| `/acp set`           | 一般的なランタイム構成オプションを書き込みます。                      | `/acp set model openai/gpt-5.2`                                |
| `/acp cwd`           | ランタイム作業ディレクトリのオーバーライドを設定します。                   | `/acp cwd /Users/user/Projects/repo`                           |
| `/acp permissions`   | 承認ポリシープロファイルを設定します。                              | `/acp permissions strict`                                      |
| `/acp timeout`       | ランタイムタイムアウト（秒）を設定します。                            | `/acp timeout 120`                                             |
| `/acp model`         | ランタイムモデルのオーバーライドを設定します。                               | `/acp model anthropic/claude-opus-4-5`                         |
| `/acp reset-options` | セッションのランタイムオプションのオーバーライドを削除します。                  | `/acp reset-options`                                           |
| `/acp sessions`      | ストアから最近の ACP セッションをリストします。                      | `/acp sessions`                                                |
| `/acp doctor`        | バックエンドの健康状態、機能、実行可能な修正を表示します。           | `/acp doctor`                                                  |
| `/acp install`       | 決定論的なインストールおよび有効化手順を表示します。             | `/acp install`                                                 |

## ランタイムオプションマッピング

`/acp` には便利なコマンドと一般的なセッターがあります。

同等の操作：

* `/acp model <id>` はランタイム構成キー `model` にマッピングされます。
* `/acp permissions <profile>` はランタイム構成キー `approval_policy` にマッピングされます。
* `/acp timeout <seconds>` はランタイム構成キー `timeout` にマッピングされます。
* `/acp cwd <path>` はランタイム cwd オーバーライドを直接更新します。
* `/acp set <key> <value>` は一般的なパスです。
  * 特殊ケース：`key=cwd` は cwd オーバーライドパスを使用します。
* `/acp reset-options` はターゲットセッションのすべてのランタイムオーバーライドをクリアします。

## acpx ハーネスサポート（現在）

現在の acpx ビルトインハーネスエイリアス：

* `pi`
* `claude`
* `codex`
* `opencode`
* `gemini`

OpenClaw が acpx バックエンドを使用する場合、カスタムエージェントエイリアスが定義されていない限り、これらの値を `agentId` に優先して使用してください。

直接的な acpx CLI 使用は `--agent <command>` を介して任意のアダプタをターゲットにすることもできますが、その生のエスケープハッチは acpx CLI 機能（通常の OpenClaw `agentId` パスではありません）です。

## 必要な設定

コア ACP ベースライン：

```json5
{
  acp: {
    enabled: true,
    dispatch: { enabled: true },
    backend: "acpx",
    defaultAgent: "codex",
    allowedAgents: ["pi", "claude", "codex", "opencode", "gemini"],
    maxConcurrentSessions: 8,
    stream: {
      coalesceIdleMs: 300,
      maxChunkChars: 1200,
    },
    runtime: {
      ttlMinutes: 120,
    },
  },
}
```

スレッドバインディング設定はチャネルアダプタ固有です。Discord の例：

```json5
{
  session: {
    threadBindings: {
      enabled: true,
      idleHours: 24,
      maxAgeHours: 0,
    },
  },
  channels: {
    discord: {
      threadBindings: {
        enabled: true,
        spawnAcpSessions: true,
      },
    },
  },
}
```

スレッドバウンド ACP スポーンが機能しない場合は、まずアダプタ機能フラグを確認してください：

* Discord: `channels.discord.threadBindings.spawnAcpSessions=true`

[Configuration Reference](/gateway/configuration-reference) を参照してください。

## acpx バックエンドのプラグイン設定

プラグインをインストールして有効にします：

```bash
openclaw plugins install @openclaw/acpx
openclaw config set plugins.entries.acpx.enabled true
```

開発中のローカルワークスペースインストール：

```bash
openclaw plugins install ./extensions/acpx
```

その後、バックエンドの健康状態を確認します：

```text
/acp doctor
```

### ピン留めされた acpx インストール戦略（現在の動作）

`@openclaw/acpx` は現在、厳格なプラグインローカルピン留めモデルを強制します：

1. 拡張機能は `extensions/acpx/package.json` に正確な acpx 依存関係をピン留めします。
2. ランタイムコマンドはプラグインローカルバイナリ（`extensions/acpx/node_modules/.bin/acpx`）に固定され、グローバル `PATH` ではありません。
3. プラグイン設定は `command` または `commandArgs` を公開しないため、ランタイムコマンドのドリフトがブロックされます。
4. スタートアップは ACP バックエンドを即座に未準備として登録します。
5. バックグラウンドの確認ジョブが `acpx --version` をピン留めされたバージョンと照合します。
6. 不足または不一致の場合、プラグインローカルインストールを実行し（`npm install --omit=dev --no-save acpx@<pinned>`）、健康状態を再確認します。

注意：

* OpenClaw のスタートアップは acpx 確認が実行されている間、ブロッキングされません。
* ネットワーク/インストールが失敗した場合、バックエンドは利用できなくなり、`/acp doctor` は実行可能な修正を報告します。

[Plugins](/tools/plugin) を参照してください。

## トラブルシューティング

| 症状                                                                 | 考えられる原因                                   | 修正                                                        |
| ----------------------------------------------------------------------- | ---------------------------------------------- | ---------------------------------------------------------- |
| `ACP runtime backend is not configured`                                 | バックエンドプラグインが欠落または無効。            | バックエンドプラグインをインストールして有効にし、`/acp doctor` を実行します。 |
| `ACP is disabled by policy (acp.enabled=false)`                         | ACP がグローバルに無効。                         | `acp.enabled=true` を設定します。                                    |
| `ACP dispatch is disabled by policy (acp.dispatch.enabled=false)`       | 通常のスレッドメッセージからのディスパッチが無効。 | `acp.dispatch.enabled=true` を設定します。                           |
| `ACP agent "<id>" is not allowed by policy`                             | エージェントが許可リストにない。                        | 許可された `agentId` を使用するか、`acp.allowedAgents` を更新します。       |
| `Unable to resolve session target: ...`                                 | 不正なキー/ID/ラベルトークン。                        | `/acp sessions` を実行し、正確なキー/ラベルをコピーして再試行します。          |
| `--thread here requires running /acp spawn inside an active ... thread` | `--thread here` がスレッドコンテキスト外で使用された。 | ターゲットスレッドに移動するか、`--thread auto`/`off` を使用します。        |
| `Only <user-id> can rebind this thread.`                                | 別のユーザーがスレッドバインディングを所有している。              | 所有者として再バインドするか、別のスレッドを使用します。                 |
| `Thread bindings are unavailable for <channel>.`                        | アダプタがスレッドバインディング機能を欠いている。       | `--thread off` を使用するか、サポートされているアダプタ/チャネルに移動します。   |
| バインドされたセッションの ACP メタデータが欠落している                                  | 古い/削除された ACP セッションメタデータ。            | `/acp spawn` で再作成し、その後再バインド/フォーカススレッドを行います。      |
