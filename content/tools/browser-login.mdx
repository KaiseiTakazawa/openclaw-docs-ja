# ブラウザログイン

# Browser login + X/Twitter posting

## 手動ログイン（推奨）

サイトがログインを要求する場合は、**ホスト**ブラウザプロファイル（openclawブラウザ）で**手動でサインイン**してください。

モデルにあなたの認証情報を提供しないでください。自動ログインはしばしばボット対策を引き起こし、アカウントがロックされる可能性があります。

メインのブラウザドキュメントに戻る: [Browser](/tools/browser)。

## どのChromeプロファイルが使用されているか？

OpenClawは**専用のChromeプロファイル**（`openclaw`という名前、オレンジ色のUI）を制御します。これはあなたの日常のブラウザプロファイルとは別です。

アクセスするための簡単な方法は2つあります：

1. **エージェントにブラウザを開くように頼み**、自分でログインします。
2. **CLI経由で開く**：

```bash
openclaw browser start
openclaw browser open https://x.com
```

複数のプロファイルがある場合は、`--browser-profile <name>`を指定してください（デフォルトは`openclaw`です）。

## X/Twitter: 推奨フロー

* **読む/検索/スレッド:** **ホスト**ブラウザを使用（手動ログイン）。
* **更新を投稿:** **ホスト**ブラウザを使用（手動ログイン）。

## サンドボックス化 + ホストブラウザアクセス

サンドボックス化されたブラウザセッションは**ボット検出を引き起こす可能性が高い**です。X/Twitter（および他の厳しいサイト）では、**ホスト**ブラウザを優先してください。

エージェントがサンドボックス化されている場合、ブラウザツールはデフォルトでサンドボックスに切り替わります。ホスト制御を許可するには：

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

その後、ホストブラウザをターゲットにします：

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

または、更新を投稿するエージェントのサンドボックス化を無効にします。
