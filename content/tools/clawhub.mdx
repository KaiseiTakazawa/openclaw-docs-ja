# ClawHub

# ClawHub

ClawHubは**OpenClawのための公開スキルレジストリ**です。これは無料のサービスで、すべてのスキルは公開されており、誰でも共有や再利用が可能です。スキルは`SKILL.md`ファイル（およびサポートテキストファイル）を含むフォルダに過ぎません。ウェブアプリでスキルをブラウズするか、CLIを使用してスキルを検索、インストール、更新、公開できます。

サイト: [clawhub.ai](https://clawhub.ai)

## ClawHubとは

* OpenClawスキルのための公開レジストリ。
* スキルバンドルとメタデータのバージョン管理されたストア。
* 検索、タグ、使用信号のための発見の場。

## 仕組み

1. ユーザーがスキルバンドル（ファイル + メタデータ）を公開します。
2. ClawHubはバンドルを保存し、メタデータを解析し、バージョンを割り当てます。
3. レジストリはスキルを検索と発見のためにインデックス化します。
4. ユーザーはOpenClawでスキルをブラウズ、ダウンロード、インストールします。

## できること

* 新しいスキルや既存スキルの新しいバージョンを公開する。
* 名前、タグ、または検索によってスキルを発見する。
* スキルバンドルをダウンロードし、そのファイルを確認する。
* 悪用または安全でないスキルを報告する。
* モデレーターであれば、非表示、再表示、削除、または禁止する。

## 対象者（初心者向け）

OpenClawエージェントに新しい機能を追加したい場合、ClawHubはスキルを見つけてインストールする最も簡単な方法です。バックエンドがどのように動作するかを知る必要はありません。あなたは：

* 普通の言葉でスキルを検索することができます。
* スキルをワークスペースにインストールすることができます。
* 後で1つのコマンドでスキルを更新することができます。
* 自分のスキルを公開してバックアップすることができます。

## クイックスタート（非技術的）

1. CLIをインストールします（次のセクションを参照）。
2. 必要なものを検索します：
   * `clawhub search "calendar"`
3. スキルをインストールします：
   * `clawhub install <skill-slug>`
4. 新しいスキルを取得するために新しいOpenClawセッションを開始します。

## CLIのインストール

いずれかを選択してください：

```bash
npm i -g clawhub
```

```bash
pnpm add -g clawhub
```

## OpenClawへの適合

デフォルトでは、CLIは現在の作業ディレクトリの`./skills`にスキルをインストールします。OpenClawワークスペースが設定されている場合、`clawhub`は`--workdir`（または`CLAWHUB_WORKDIR`）を上書きしない限り、そのワークスペースにフォールバックします。OpenClawは`<workspace>/skills`からワークスペーススキルを読み込み、**次の**セッションでそれらを取得します。すでに`~/.openclaw/skills`やバンドルされたスキルを使用している場合、ワークスペーススキルが優先されます。

スキルがどのように読み込まれ、共有され、制限されるかの詳細については、[Skills](/tools/skills)を参照してください。

## スキルシステムの概要

スキルは、OpenClawに特定のタスクを実行する方法を教えるバージョン管理されたファイルのバンドルです。各公開は新しいバージョンを作成し、レジストリは変更を監査できるようにバージョンの履歴を保持します。

典型的なスキルには以下が含まれます：

* 主な説明と使用法を含む`SKILL.md`ファイル。
* スキルによって使用されるオプションの設定、スクリプト、またはサポートファイル。
* タグ、要約、インストール要件などのメタデータ。

ClawHubはメタデータを使用して発見を促進し、スキルの機能を安全に公開します。レジストリはまた、ランキングと可視性を向上させるために使用信号（スターやダウンロードなど）を追跡します。

## サービスが提供するもの（機能）

* **スキルとその`SKILL.md`コンテンツの公開ブラウジング**。
* **キーワードだけでなく、埋め込み（ベクトル検索）による検索**。
* **セマンティックバージョニング、チェンジログ、タグ（`latest`を含む）によるバージョン管理**。
* **バージョンごとのzipとしてのダウンロード**。
* **コミュニティフィードバックのためのスターとコメント**。
* **承認と監査のためのモデレーションフック**。
* **自動化とスクリプティングのためのCLIフレンドリーなAPI**。

## セキュリティとモデレーション

ClawHubはデフォルトでオープンです。誰でもスキルをアップロードできますが、GitHubアカウントは公開するために少なくとも1週間の古さが必要です。これにより、正当な貢献者をブロックすることなく、悪用を遅らせることができます。

報告とモデレーション：

* サインインしたユーザーは誰でもスキルを報告できます。
* 報告理由は必須で記録されます。
* 各ユーザーは同時に最大20件のアクティブな報告を持つことができます。
* 3件以上のユニークな報告があるスキルはデフォルトで自動的に非表示になります。
* モデレーターは非表示のスキルを表示し、それを再表示、削除、またはユーザーを禁止できます。
* 報告機能の悪用はアカウントの禁止を招く可能性があります。

モデレーターになることに興味がありますか？OpenClawのDiscordで尋ねて、モデレーターまたはメンテナに連絡してください。

## CLIコマンドとパラメータ

グローバルオプション（すべてのコマンドに適用）：

* `--workdir <dir>`: 作業ディレクトリ（デフォルト: 現在のディレクトリ; OpenClawワークスペースにフォールバック）。
* `--dir <dir>`: workdirに対する相対的なスキルディレクトリ（デフォルト: `skills`）。
* `--site <url>`: サイトのベースURL（ブラウザログイン）。
* `--registry <url>`: レジストリアPIのベースURL。
* `--no-input`: プロンプトを無効にする（非対話型）。
* `-V, --cli-version`: CLIのバージョンを表示。

認証：

* `clawhub login`（ブラウザフロー）または`clawhub login --token <token>`
* `clawhub logout`
* `clawhub whoami`

オプション：

* `--token <token>`: APIトークンを貼り付けます。
* `--label <label>`: ブラウザログイントークンに保存されるラベル（デフォルト: `CLI token`）。
* `--no-browser`: ブラウザを開かない（`--token`が必要）。

検索：

* `clawhub search "query"`
* `--limit <n>`: 最大結果数。

インストール：

* `clawhub install <slug>`
* `--version <version>`: 特定のバージョンをインストール。
* `--force`: フォルダがすでに存在する場合は上書き。

更新：

* `clawhub update <slug>`
* `clawhub update --all`
* `--version <version>`: 特定のバージョンに更新（単一のスラグのみ）。
* `--force`: ローカルファイルが公開されたバージョンと一致しない場合は上書き。

リスト：

* `clawhub list`（`.clawhub/lock.json`を読み取る）

公開：

* `clawhub publish <path>`
* `--slug <slug>`: スキルスラグ。
* `--name <name>`: 表示名。
* `--version <version>`: セマンティックバージョン。
* `--changelog <text>`: チェンジログテキスト（空でも可）。
* `--tags <tags>`: カンマ区切りのタグ（デフォルト: `latest`）。

削除/復元（所有者/管理者のみ）：

* `clawhub delete <slug> --yes`
* `clawhub undelete <slug> --yes`

同期（ローカルスキルをスキャン + 新規/更新を公開）：

* `clawhub sync`
* `--root <dir...>`: 追加のスキャンルート。
* `--all`: プロンプトなしで全てをアップロード。
* `--dry-run`: アップロードされるものを表示。
* `--bump <type>`: 更新のための`patch|minor|major`（デフォルト: `patch`）。
* `--changelog <text>`: 非対話型更新のためのチェンジログ。
* `--tags <tags>`: カンマ区切りのタグ（デフォルト: `latest`）。
* `--concurrency <n>`: レジストリチェック（デフォルト: 4）。

## エージェントの一般的なワークフロー

### スキルを検索する

```bash
clawhub search "postgres backups"
```

### 新しいスキルをダウンロードする

```bash
clawhub install my-skill-pack
```

### インストールされたスキルを更新する

```bash
clawhub update --all
```

### スキルをバックアップする（公開または同期）

単一のスキルフォルダの場合：

```bash
clawhub publish ./my-skill --slug my-skill --name "My Skill" --version 1.0.0 --tags latest
```

複数のスキルを一度にスキャンしてバックアップするには：

```bash
clawhub sync --all
```

## 詳細情報（技術的）

### バージョン管理とタグ

* 各公開は新しい**セマンティックバージョン**`SkillVersion`を作成します。
* タグ（`latest`など）はバージョンを指し、タグを移動させることでロールバックできます。
* チェンジログはバージョンごとに添付され、同期または更新を公開する際に空であっても構いません。

### ローカル変更とレジストリバージョン

更新はローカルスキルの内容をレジストリバージョンと比較し、コンテンツハッシュを使用します。ローカルファイルが公開されたバージョンと一致しない場合、CLIは上書きする前に確認を求めます（または非対話型実行では`--force`が必要です）。

### 同期スキャンとフォールバックルート

`clawhub sync`は最初に現在のworkdirをスキャンします。スキルが見つからない場合、既知のレガシー場所（例えば`~/openclaw/skills`や`~/.openclaw/skills`）にフォールバックします。これは、追加のフラグなしで古いスキルインストールを見つけるために設計されています。

### ストレージとロックファイル

* インストールされたスキルは、あなたのworkdirの`.clawhub/lock.json`に記録されます。
* 認証トークンはClawHub CLI設定ファイルに保存されます（`CLAWHUB_CONFIG_PATH`で上書き可能）。

### テレメトリ（インストール数）

ログイン中に`clawhub sync`を実行すると、CLIはインストール数を計算するための最小限のスナップショットを送信します。これを完全に無効にすることができます：

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

## 環境変数

* `CLAWHUB_SITE`: サイトURLを上書きします。
* `CLAWHUB_REGISTRY`: レジストリアPIのURLを上書きします。
* `CLAWHUB_CONFIG_PATH`: CLIがトークン/設定を保存する場所を上書きします。
* `CLAWHUB_WORKDIR`: デフォルトのworkdirを上書きします。
* `CLAWHUB_DISABLE_TELEMETRY=1`: `sync`でのテレメトリを無効にします。
