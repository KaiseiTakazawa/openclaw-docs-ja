# CI パイプライン
CIは`main`へのすべてのプッシュとすべてのプルリクエストで実行されます。ドキュメントやネイティブコードのみが変更された場合、高価なジョブをスキップするためにスマートスコーピングを使用します。
## ジョブの概要
| ジョブ | 目的 | 実行タイミング |
| ----------------- | ----------------------------------------------- | ------------------------- |
| `docs-scope` | ドキュメントのみの変更を検出 | いつでも |
| `changed-scope` | どの領域が変更されたかを検出（node/macos/android） | 非ドキュメントPR |
| `check` | TypeScriptの型、リント、フォーマット | 非ドキュメントの変更 |
| `check-docs` | Markdownリント + 壊れたリンクチェック | ドキュメントが変更されたとき |
| `code-analysis` | LOCしきい値チェック（1000行） | PRのみに対して |
| `secrets` | 漏洩した秘密を検出 | いつでも |
| `build-artifacts` | 一度だけdistをビルドし、他のジョブと共有 | 非ドキュメント、nodeの変更 |
| `release-check` | npmパックの内容を検証 | ビルド後 |
| `checks` | Node/Bunテスト + プロトコルチェック | 非ドキュメント、nodeの変更 |
| `checks-windows` | Windows特有のテスト | 非ドキュメント、nodeの変更 |
| `macos` | Swiftリント/ビルド/テスト + TSテスト | macosの変更を含むPR |
| `android` | Gradleビルド + テスト | 非ドキュメント、androidの変更 |
## フェイルファスト順序
ジョブは、安価なチェックが高価なものが実行される前に失敗するように順序付けられています：
1. `docs-scope` + `code-analysis` + `check`（並列、約1-2分）
2. `build-artifacts`（上記にブロックされる）
3. `checks`、`checks-windows`、`macos`、`android`（ビルドにブロックされる）
## ランナー
| ランナー | ジョブ |
| -------------------------------- | ------------------------------------------ |
| `blacksmith-16vcpu-ubuntu-2404` | ほとんどのLinuxジョブ、スコープ検出を含む |
| `blacksmith-16vcpu-windows-2025` | `checks-windows` |
| `macos-latest` | `macos`、`ios` |
## ローカルの同等物
```bash
pnpm check # 型 + リント + フォーマット
pnpm test # vitestテスト
pnpm check:docs # ドキュメントフォーマット + リント + 壊れたリンク
pnpm release:check # npmパックを検証
```
