# コンパクション

# コンテキストウィンドウとコンパクション

すべてのモデルには**コンテキストウィンドウ**（モデルが見ることができる最大トークン数）があります。長時間のチャットではメッセージやツールの結果が蓄積され、ウィンドウが狭くなると、OpenClawは**コンパクト**にするために古い履歴を圧縮します。

## コンパクションとは

コンパクションは**古い会話を要約**し、コンパクトな要約エントリを作成し、最近のメッセージはそのまま保持します。要約はセッションの履歴に保存されるため、将来のリクエストでは以下を使用します：

* コンパクション要約
* コンパクションポイント以降の最近のメッセージ

コンパクションはセッションのJSONL履歴に**永続化**されます。

## 設定

`openclaw.json`の`agents.defaults.compaction`設定を使用して、コンパクションの動作（モード、ターゲットトークンなど）を構成します。
コンパクションの要約はデフォルトで不透明な識別子を保持します（`identifierPolicy: "strict"`）。これを`identifierPolicy: "off"`でオーバーライドするか、`identifierPolicy: "custom"`と`identifierInstructions`を使用してカスタムテキストを提供できます。

## 自動コンパクション（デフォルトオン）

セッションがモデルのコンテキストウィンドウに近づくか、超過すると、OpenClawは自動コンパクションをトリガーし、圧縮されたコンテキストを使用して元のリクエストを再試行することがあります。

以下が表示されます：

* 詳細モードで`🧹 自動コンパクション完了`
* `/status`で`🧹 コンパクション: <count>`

コンパクションの前に、OpenClawは**サイレントメモリフラッシュ**ターンを実行して、耐久性のあるノートをディスクに保存できます。詳細と設定については[Memory](/concepts/memory)を参照してください。

## 手動コンパクション

`/compact`（オプションで指示付き）を使用して、コンパクションパスを強制します：

```
/compact 決定事項と未解決の質問に焦点を当てる
```

## コンテキストウィンドウのソース

コンテキストウィンドウはモデル固有です。OpenClawは、構成されたプロバイダカタログからモデル定義を使用して制限を決定します。

## コンパクションとプルーニングの違い

* **コンパクション**: 要約し、JSONLに**永続化**します。
* **セッションプルーニング**: 古い**ツール結果**のみを**メモリ内**でリクエストごとにトリムします。

プルーニングの詳細については[/concepts/session-pruning](/concepts/session-pruning)を参照してください。

## OpenAIサーバーサイドコンパクション

OpenClawは、互換性のある直接OpenAIモデルのためのOpenAI Responsesサーバーサイドコンパクションヒントもサポートしています。これはローカルのOpenClawコンパクションとは別で、同時に実行できます。

* ローカルコンパクション: OpenClawが要約し、セッションJSONLに永続化します。
* サーバーサイドコンパクション: OpenAIがプロバイダ側でコンテキストを圧縮します（`store` + `context_management`が有効な場合）。

モデルパラメータとオーバーライドについては[OpenAI provider](/providers/openai)を参照してください。

## ヒント

* セッションが古く感じたり、コンテキストが膨らんでいる場合は`/compact`を使用してください。
* 大きなツール出力はすでに切り捨てられています。プルーニングはツール結果の蓄積をさらに減少させることができます。
* 新しいスレートが必要な場合は、`/new`または`/reset`で新しいセッションIDを開始します。
