# マルチゲートウェイ

# Multiple Gateways (same host)

ほとんどのセットアップでは、1つのGatewayを使用するべきです。なぜなら、単一のGatewayが複数のメッセージング接続とエージェントを処理できるからです。より強い隔離や冗長性（例：救助ボット）が必要な場合は、隔離されたプロファイル/ポートを持つ別々のGatewaysを実行してください。

## Isolation checklist (required)

* `OPENCLAW_CONFIG_PATH` — インスタンスごとの設定ファイル
* `OPENCLAW_STATE_DIR` — インスタンスごとのセッション、クレデンシャル、キャッシュ
* `agents.defaults.workspace` — インスタンスごとのワークスペースルート
* `gateway.port` (または `--port`) — インスタンスごとにユニーク
* 派生ポート（ブラウザ/キャンバス）は重複してはいけません

これらが共有されると、設定競合やポートの衝突が発生します。

## Recommended: profiles (`--profile`)

プロファイルは `OPENCLAW_STATE_DIR` と `OPENCLAW_CONFIG_PATH` を自動的にスコープし、サービス名にサフィックスを付けます。

```bash
# main
openclaw --profile main setup
openclaw --profile main gateway --port 18789

# rescue
openclaw --profile rescue setup
openclaw --profile rescue gateway --port 19001
```

プロファイルごとのサービス:

```bash
openclaw --profile main gateway install
openclaw --profile rescue gateway install
```

## Rescue-bot guide

同じホスト上で、独自の以下を持つ第二のGatewayを実行します：

* プロファイル/設定
* ステートディレクトリ
* ワークスペース
* ベースポート（および派生ポート）

これにより、救助ボットはメインボットから隔離され、プライマリボットがダウンしている場合にデバッグや設定変更を適用できます。

ポートの間隔：ベースポートの間に少なくとも20ポートを空けて、派生ブラウザ/キャンバス/CDPポートが衝突しないようにします。

### How to install (rescue bot)

```bash
# Main bot (既存または新規、--profileパラメータなし)
# ポート18789 + Chrome CDC/Canvas/... ポートで実行
openclaw onboard
openclaw gateway install

# Rescue bot (隔離されたプロファイル + ポート)
openclaw --profile rescue onboard
# 注意：
# - ワークスペース名はデフォルトで -rescue が付加されます
# - ポートは少なくとも18789 + 20ポートである必要があります、
#   完全に異なるベースポート（例：19789）を選択する方が良いです、
# - 残りのオンボーディングは通常と同じです

# サービスをインストールするために（オンボーディング中に自動的に行われなかった場合）
openclaw --profile rescue gateway install
```

## Port mapping (derived)

ベースポート = `gateway.port`（または `OPENCLAW_GATEWAY_PORT` / `--port`）。

* ブラウザ制御サービスポート = ベース + 2（ループバック専用）
* キャンバスホストはGateway HTTPサーバー上で提供されます（`gateway.port`と同じポート）
* ブラウザプロファイルCDPポートは `browser.controlPort + 9 .. + 108` から自動割り当てされます

これらのいずれかを設定または環境変数で上書きする場合、インスタンスごとにユニークである必要があります。

## Browser/CDP notes (common footgun)

* 複数のインスタンスで `browser.cdpUrl` を同じ値に固定しないでください。
* 各インスタンスには独自のブラウザ制御ポートとCDP範囲が必要です（そのGatewayポートから派生）。
* 明示的なCDPポートが必要な場合は、インスタンスごとに `browser.profiles.<name>.cdpPort` を設定してください。
* リモートChrome：`browser.profiles.<name>.cdpUrl` を使用します（プロファイルごと、インスタンスごと）。

## Manual env example

```bash
OPENCLAW_CONFIG_PATH=~/.openclaw/main.json \
OPENCLAW_STATE_DIR=~/.openclaw-main \
openclaw gateway --port 18789

OPENCLAW_CONFIG_PATH=~/.openclaw/rescue.json \
OPENCLAW_STATE_DIR=~/.openclaw-rescue \
openclaw gateway --port 19001
```

## Quick checks

```bash
openclaw --profile main status
openclaw --profile rescue status
openclaw --profile rescue browser status
```
