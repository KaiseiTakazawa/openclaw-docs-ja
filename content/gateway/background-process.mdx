# バックグラウンド実行とプロセスツール

# Background Exec + Process Tool

OpenClawは`exec`ツールを介してシェルコマンドを実行し、長時間実行されるタスクをメモリ内に保持します。`process`ツールはこれらのバックグラウンドセッションを管理します。

## execツール

主なパラメータ:

* `command` (必須)
* `yieldMs` (デフォルト10000): この遅延後に自動的にバックグラウンド
* `background` (bool): すぐにバックグラウンド
* `timeout` (秒, デフォルト1800): このタイムアウト後にプロセスを終了
* `elevated` (bool): 高度なモードが有効/許可されている場合、ホストで実行
* 実際のTTYが必要ですか？`pty: true`を設定します。
* `workdir`, `env`

動作:

* フォアグラウンド実行は出力を直接返します。
* バックグラウンドにされた場合（明示的またはタイムアウト）、ツールは`status: "running"` + `sessionId`と短いテールを返します。
* 出力はセッションがポーリングされるかクリアされるまでメモリ内に保持されます。
* `process`ツールが許可されていない場合、`exec`は同期的に実行され、`yieldMs`/`background`を無視します。

## 子プロセスブリッジ

`exec`/`process`ツールの外で長時間実行される子プロセスを生成する場合（例えば、CLIの再生成やゲートウェイヘルパー）、子プロセスブリッジヘルパーを接続して、終了シグナルが転送され、リスナーが終了/エラー時にデタッチされるようにします。これにより、systemd上での孤児プロセスを回避し、プラットフォーム間でシャットダウン動作を一貫させます。

環境オーバーライド:

* `PI_BASH_YIELD_MS`: デフォルトのイールド（ms）
* `PI_BASH_MAX_OUTPUT_CHARS`: メモリ内出力の上限（文字数）
* `OPENCLAW_BASH_PENDING_MAX_OUTPUT_CHARS`: ストリームごとの保留中のstdout/stderrの上限（文字数）
* `PI_BASH_JOB_TTL_MS`: 完了したセッションのTTL（ms、1m〜3hに制限）

設定（推奨）:

* `tools.exec.backgroundMs` (デフォルト10000)
* `tools.exec.timeoutSec` (デフォルト1800)
* `tools.exec.cleanupMs` (デフォルト1800000)
* `tools.exec.notifyOnExit` (デフォルトtrue): バックグラウンドで実行されたexecが終了したときにシステムイベントをキューに追加し、ハートビートを要求します。
* `tools.exec.notifyOnExitEmptySuccess` (デフォルトfalse): trueの場合、出力を生成しなかった成功したバックグラウンド実行の完了イベントもキューに追加します。

## processツール

アクション:

* `list`: 実行中 + 完了したセッション
* `poll`: セッションの新しい出力を排出（終了ステータスも報告）
* `log`: 集約された出力を読み取る（`offset` + `limit`をサポート）
* `write`: stdinを送信（`data`, オプションで`eof`）
* `kill`: バックグラウンドセッションを終了
* `clear`: メモリから完了したセッションを削除
* `remove`: 実行中の場合は終了、そうでなければ完了している場合はクリア

注意:

* バックグラウンドにされたセッションのみがリストされ、メモリに保持されます。
* プロセス再起動時にセッションは失われます（ディスク持続性なし）。
* セッションログは、`process poll/log`を実行し、ツールの結果が記録される場合にのみチャット履歴に保存されます。
* `process`はエージェントごとにスコープされており、そのエージェントによって開始されたセッションのみを表示します。
* `process list`には、迅速なスキャンのために派生した`name`（コマンド動詞 + 対象）が含まれます。
* `process log`は行ベースの`offset`/`limit`を使用します。
* `offset`と`limit`の両方が省略された場合、最後の200行を返し、ページングのヒントを含みます。
* `offset`が提供され、`limit`が省略された場合、`offset`から最後までを返します（200に制限されません）。

## 例

長いタスクを実行し、後でポーリングします:

```json
{ "tool": "exec", "command": "sleep 5 && echo done", "yieldMs": 1000 }
```

```json
{ "tool": "process", "action": "poll", "sessionId": "<id>" }
```

バックグラウンドで即座に開始:

```json
{ "tool": "exec", "command": "npm run build", "background": true }
```

stdinを送信:

```json
{ "tool": "process", "action": "write", "sessionId": "<id>", "data": "y\n" }
```
