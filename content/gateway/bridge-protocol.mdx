# ブリッジプロトコル

# Bridge protocol (legacy node transport)

ブリッジプロトコルは**レガシー**ノードトランスポート（TCP JSONL）です。新しいノードクライアントは、代わりに統一されたGateway WebSocketプロトコルを使用するべきです。

オペレーターまたはノードクライアントを構築している場合は、[Gatewayプロトコル](/gateway/protocol)を使用してください。

**注意:** 現在のOpenClawビルドにはTCPブリッジリスナーは含まれていません。この文書は歴史的な参考のために保持されています。レガシー`bridge.*`設定キーはもはや設定スキーマの一部ではありません。

## なぜ両方が必要なのか

* **セキュリティ境界**: ブリッジは完全なゲートウェイAPIサーフェスの代わりに、小さな許可リストを公開します。
* **ペアリング + ノードアイデンティティ**: ノードの入場はゲートウェイによって所有され、ノードごとのトークンに結びついています。
* **ディスカバリUX**: ノードはLAN上でBonjourを介してゲートウェイを発見するか、tailnetを介して直接接続できます。
* **ループバックWS**: 完全なWSコントロールプレーンは、SSHを介ってトンネルされない限りローカルに留まります。

## トランスポート

* TCP、1行につき1つのJSONオブジェクト（JSONL）。
* オプションのTLS（`bridge.tls.enabled`がtrueの場合）。
* レガシーのデフォルトリスナーポートは`18790`でした（現在のビルドはTCPブリッジを起動しません）。

TLSが有効な場合、ディスカバリTXTレコードには`bridgeTls=1`と`bridgeTlsSha256`が含まれます。Bonjour/mDNS TXTレコードは認証されていないため、クライアントは広告されたフィンガープリントを明示的なユーザーの意図や他のアウトオブバンド確認なしに権威あるピンとして扱うべきではありません。

## ハンドシェイク + ペアリング

1. クライアントはノードメタデータ + トークン（すでにペアリングされている場合）を含む`hello`を送信します。
2. ペアリングされていない場合、ゲートウェイは`error`（`NOT_PAIRED`/`UNAUTHORIZED`）で応答します。
3. クライアントは`pair-request`を送信します。
4. ゲートウェイは承認を待ち、次に`pair-ok`と`hello-ok`を送信します。

`hello-ok`は`serverName`を返し、`canvasHostUrl`を含む場合があります。

## フレーム

クライアント → ゲートウェイ:

* `req` / `res`: スコープ付きゲートウェイRPC（チャット、セッション、設定、ヘルス、ボイスウェイク、skills.bins）
* `event`: ノードシグナル（音声トランスクリプト、エージェントリクエスト、チャット購読、execライフサイクル）

ゲートウェイ → クライアント:

* `invoke` / `invoke-res`: ノードコマンド（`canvas.*`、`camera.*`、`screen.record`、`location.get`、`sms.send`）
* `event`: 購読されたセッションのチャット更新
* `ping` / `pong`: キープアライブ

レガシーの許可リストの強制は`src/gateway/server-bridge.ts`に存在しました（削除済み）。

## Execライフサイクルイベント

ノードは`exec.finished`または`exec.denied`イベントを発信してsystem.runアクティビティを表示できます。これらはゲートウェイ内のシステムイベントにマッピングされます。（レガシーノードはまだ`exec.started`を発信する場合があります。）

ペイロードフィールド（特に記載がない限りすべてオプション）:

* `sessionKey`（必須）: システムイベントを受信するエージェントセッション。
* `runId`: グループ化のためのユニークなexec ID。
* `command`: 生またはフォーマットされたコマンド文字列。
* `exitCode`、`timedOut`、`success`、`output`: 完了の詳細（終了時のみ）。
* `reason`: 拒否理由（拒否時のみ）。

## Tailnetの使用

* ブリッジをtailnet IPにバインド: `bridge.bind: "tailnet"`を`~/.openclaw/openclaw.json`に記述。
* クライアントはMagicDNS名またはtailnet IPを介して接続します。
* Bonjourは**ネットワークを越えません**; 必要に応じて手動のホスト/ポートまたは広域DNS‑SDを使用してください。

## バージョニング

ブリッジは現在**暗黙のv1**（最小/最大交渉なし）です。後方互換性が期待されます。破壊的変更の前にブリッジプロトコルバージョンフィールドを追加してください。
