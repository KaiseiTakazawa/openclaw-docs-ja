# 自動化トラブルシューティング
# Automation troubleshooting
このページはスケジューラーと配信の問題（`cron` + `heartbeat`）に使用します。
## コマンドラダー
```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```
次に自動化チェックを実行します：
```bash
openclaw cron status
openclaw cron list
openclaw system heartbeat last
```
## Cronが発火しない
```bash
openclaw cron status
openclaw cron list
openclaw cron runs --id  --limit 20
openclaw logs --follow
```
良い出力は次のようになります：
\* `cron status` が有効であり、将来の `nextWakeAtMs` を報告します。
\* ジョブは有効で、正しいスケジュール/タイムゾーンがあります。
\* `cron runs` が `ok` または明示的なスキップ理由を示します。
一般的なサイン：
\* `cron: scheduler disabled; jobs will not run automatically` → 設定/環境でcronが無効になっています。
\* `cron: timer tick failed` → スケジューラーのティックがクラッシュしました；周囲のスタック/ログコンテキストを確認してください。
\* 実行出力に `reason: not-due` → `--force`なしで手動実行が呼び出され、ジョブがまだ期日ではありません。
## Cronが発火したが配信がない
```bash
openclaw cron runs --id  --limit 20
openclaw cron list
openclaw channels status --probe
openclaw logs --follow
```
良い出力は次のようになります：
\* 実行ステータスは `ok` です。
\* 配信モード/ターゲットは孤立したジョブに設定されています。
\* チャンネルプローブがターゲットチャンネルに接続されていることを報告します。
一般的なサイン：
\* 実行は成功したが配信モードが `none` → 外部メッセージは期待されていません。
\* 配信ターゲットが欠落/無効（`channel`/`to`）→ 実行は内部で成功するかもしれませんが、外向きはスキップされます。
\* チャンネル認証エラー（`unauthorized`, `missing_scope`, `Forbidden`）→ チャンネルの資格情報/権限によって配信がブロックされました。
## ハートビートが抑制またはスキップされた
```bash
openclaw system heartbeat last
openclaw logs --follow
openclaw config get agents.defaults.heartbeat
openclaw channels status --probe
```
良い出力は次のようになります：
\* ハートビートが有効で、ゼロ以外の間隔があります。
\* 最後のハートビート結果は `ran`（またはスキップ理由が理解されています）。
一般的なサイン：
\* `heartbeat skipped` と `reason=quiet-hours` → `activeHours` の外。
\* `requests-in-flight` → メインレーンがビジー；ハートビートが保留されました。
\* `empty-heartbeat-file` → インターバルハートビートがスキップされたのは `HEARTBEAT.md` に実行可能なコンテンツがなく、タグ付きのcronイベントがキューにないためです。
\* `alerts-disabled` → 可視性設定が外向きのハートビートメッセージを抑制します。
## タイムゾーンと activeHours の注意点
```bash
openclaw config get agents.defaults.heartbeat.activeHours
openclaw config get agents.defaults.heartbeat.activeHours.timezone
openclaw config get agents.defaults.userTimezone || echo "agents.defaults.userTimezone not set"
openclaw cron list
openclaw logs --follow
```
簡単なルール：
\* `Config path not found: agents.defaults.userTimezone` はキーが設定されていないことを意味します；ハートビートはホストのタイムゾーンにフォールバックします（または設定されている場合は `activeHours.timezone`）。
\* `--tz`なしのcronはゲートウェイホストのタイムゾーンを使用します。
\* ハートビートの `activeHours` は構成されたタイムゾーン解決（`user`, `local`, または明示的なIANA tz）を使用します。
\* タイムゾーンなしのISOタイムスタンプは、cronの `at` スケジュールのためにUTCとして扱われます。
一般的なサイン：
\* ホストのタイムゾーンが変更された後、ジョブが間違った壁時計時間で実行されます。
\* あなたの昼間にハートビートが常にスキップされるのは、`activeHours.timezone` が間違っているからです。
