# security

# `openclaw security`

セキュリティツール（監査 + オプションの修正）。

関連:

* セキュリティガイド: [Security](/gateway/security)

## 監査

```bash
openclaw security audit
openclaw security audit --deep
openclaw security audit --fix
openclaw security audit --json
```

監査は、複数のDM送信者がメインセッションを共有している場合に警告を発し、**セキュアDMモード**を推奨します: `session.dmScope="per-channel-peer"`（またはマルチアカウントチャネルの場合は`per-account-channel-peer`）は共有受信トレイ用です。
これは協力的/共有受信トレイの強化のためです。相互に信頼できない/対立するオペレーターによって共有される単一のGatewayは推奨される設定ではありません。別々のGateway（または別々のOSユーザー/ホスト）で信頼境界を分割してください。
また、設定が共有ユーザーの侵入を示唆する場合（例えば、オープンDM/グループポリシー、構成されたグループターゲット、またはワイルドカード送信者ルールなど）、`security.trust_model.multi_user_heuristic`を発信し、OpenClawはデフォルトでパーソナルアシスタントの信頼モデルであることを思い出させます。
意図的な共有ユーザー設定の場合、監査のガイダンスはすべてのセッションをサンドボックス化し、ファイルシステムアクセスをワークスペーススコープに保ち、個人/プライベートのアイデンティティや資格情報をそのランタイムから外すことです。
また、小さなモデル（`<=300B`）がサンドボックス化されず、ウェブ/ブラウザツールが有効になっている場合に警告します。
Webhookの侵入については、`hooks.defaultSessionKey`が未設定の場合、リクエストの`sessionKey`オーバーライドが有効な場合、`hooks.allowedSessionKeyPrefixes`なしでオーバーライドが有効な場合に警告します。
また、サンドボックスモードがオフの状態でサンドボックスDocker設定が構成されている場合、`gateway.nodes.denyCommands`が効果のないパターンのような/未知のエントリを使用している場合（正確なノードコマンド名のマッチングのみ、シェルテキストフィルタリングではない）、`gateway.nodes.allowCommands`が危険なノードコマンドを明示的に有効にしている場合、グローバルな`tools.profile="minimal"`がエージェントツールプロファイルによってオーバーライドされている場合、オープングループがランタイム/ファイルシステムツールをサンドボックス/ワークスペースガードなしで公開している場合、インストールされた拡張プラグインツールが許可されたツールポリシーの下でアクセス可能な場合に警告します。
また、`gateway.allowRealIpFallback=true`（プロキシが誤設定された場合のヘッダー詐称リスク）および`discovery.mdns.mode="full"`（mDNS TXTレコードを介したメタデータ漏洩）をフラグします。
サンドボックスブラウザがDocker `bridge`ネットワークを使用しているが、`sandbox.browser.cdpSourceRange`が設定されていない場合にも警告します。
危険なサンドボックスDockerネットワークモード（`host`および`container:*`名前空間の結合を含む）にもフラグを立てます。
既存のサンドボックスブラウザDockerコンテナに欠落または古いハッシュラベルがある場合（例えば、移行前のコンテナが`openclaw.browserConfigEpoch`を欠いている場合）にも警告し、`openclaw sandbox recreate --browser --all`を推奨します。
npmベースのプラグイン/フックインストール記録がピン留めされていない、整合性メタデータが欠落している、または現在インストールされているパッケージバージョンからずれている場合にも警告します。
チャネルの許可リストが安定したIDの代わりに可変の名前/メールアドレス/タグに依存している場合（Discord、Slack、Google Chat、MS Teams、Mattermost、IRCスコープが適用される場合）にも警告します。
`gateway.auth.mode="none"`がGateway HTTP APIを共有シークレットなしでアクセス可能にしている場合にも警告します（`/tools/invoke`および有効な`/v1/*`エンドポイント）。
`dangerous`/`dangerously`で始まる設定は明示的なブレークグラスオペレーターオーバーライドです；一つを有効にすること自体は、セキュリティ脆弱性の報告ではありません。
完全な危険パラメータのインベントリについては、[Security](/gateway/security)の「不安全または危険なフラグの概要」セクションを参照してください。

## JSON出力

CI/ポリシーチェックには`--json`を使用します：

```bash
openclaw security audit --json | jq '.summary'
openclaw security audit --deep --json | jq '.findings[] | select(.severity=="critical") | .checkId'
```

`--fix`と`--json`を組み合わせると、出力には修正アクションと最終報告の両方が含まれます：

```bash
openclaw security audit --fix --json | jq '{fix: .fix.ok, summary: .report.summary}'
```

## `--fix`が変更する内容

`--fix`は安全で決定論的な修正を適用します：

* 一般的な`groupPolicy="open"`を`groupPolicy="allowlist"`に変更します（サポートされているチャネル内のアカウントバリアントを含む）
* `logging.redactSensitive`を`"off"`から`"tools"`に設定します
* 状態/設定および一般的な機密ファイル（`credentials/*.json`、`auth-profiles.json`、`sessions.json`、セッション`*.jsonl`）の権限を厳しくします

`--fix`は**しません**：

* トークン/パスワード/APIキーを回転させる
* ツールを無効にする（`gateway`、`cron`、`exec`など）
* Gatewayのバインド/認証/ネットワーク露出の選択を変更する
* プラグイン/スキルを削除または書き換える
