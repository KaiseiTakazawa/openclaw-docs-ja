# cron

# `openclaw cron`

Gatewayスケジューラーのためのcronジョブを管理します。

関連:

* Cronジョブ: [Cron jobs](/automation/cron-jobs)

ヒント: 完全なコマンドのサーフェスを表示するには `openclaw cron --help` を実行してください。

注意: 孤立した `cron add` ジョブはデフォルトで `--announce` 配信になります。出力を内部に保持するには `--no-deliver` を使用してください。 `--deliver` は `--announce` の非推奨エイリアスとして残ります。

注意: ワンショット (`--at`) ジョブは成功後にデフォルトで削除されます。保持するには `--keep-after-run` を使用してください。

注意: 繰り返しジョブは、連続エラーの後に指数バックオフを使用します（30秒 → 1分 → 5分 → 15分 → 60分）、次の成功した実行の後に通常のスケジュールに戻ります。

注意: 保持/剪定は設定で制御されます：

* `cron.sessionRetention`（デフォルト `24h`）は完了した孤立実行セッションを剪定します。
* `cron.runLog.maxBytes` + `cron.runLog.keepLines` は `~/.openclaw/cron/runs/<jobId>.jsonl` を剪定します。

## 一般的な編集

メッセージを変更せずに配信設定を更新します：

```bash
openclaw cron edit <job-id> --announce --channel telegram --to "123456789"
```

孤立したジョブの配信を無効にします：

```bash
openclaw cron edit <job-id> --no-deliver
```

特定のチャンネルにアナウンスします：

```bash
openclaw cron edit <job-id> --announce --channel slack --to "channel:C1234567890"
```
