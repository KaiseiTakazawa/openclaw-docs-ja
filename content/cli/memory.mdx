# memory

# `openclaw memory`

セマンティックメモリのインデックス作成と検索を管理します。アクティブメモリプラグイン（デフォルト: `memory-core`）によって提供されます。無効にするには `plugins.slots.memory = "none"` を設定してください。

関連情報:

* メモリの概念: [Memory](/concepts/memory)
* プラグイン: [Plugins](/tools/plugin)

## 例

```bash
openclaw memory status
openclaw memory status --deep
openclaw memory status --deep --index
openclaw memory status --deep --index --verbose
openclaw memory index
openclaw memory index --verbose
openclaw memory search "release checklist"
openclaw memory search --query "release checklist"
openclaw memory status --agent main
openclaw memory index --agent main --verbose
```

## オプション

共通:

* `--agent <id>`: 単一のエージェントにスコープを絞る（デフォルト: 設定されたすべてのエージェント）。
* `--verbose`: プローブとインデックス作成中に詳細なログを出力します。

`memory search`:

* クエリ入力: 位置引数 `[query]` または `--query <text>` のいずれかを渡します。
* 両方が提供された場合、`--query` が優先されます。
* どちらも提供されない場合、コマンドはエラーで終了します。

注意:

* `memory status --deep` はベクトル + 埋め込みの可用性をプローブします。
* `memory status --deep --index` はストアが汚れている場合に再インデックスを実行します。
* `memory index --verbose` はフェーズごとの詳細（プロバイダー、モデル、ソース、バッチアクティビティ）を出力します。
* `memory status` は `memorySearch.extraPaths` を介して設定された追加パスを含みます。
