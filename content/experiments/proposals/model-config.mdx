# Model Config (Exploration)

このドキュメントは、将来のモデル設定に関する**アイデア**をまとめたものです。これは出荷用の仕様書ではありません。現在の動作については、以下を参照してください。

* [Models](/concepts/models)
* [Model failover](/concepts/model-failover)
* [OAuth + profiles](/concepts/oauth)

## 動機

オペレーターは以下を望んでいます：

* プロバイダーごとの複数の認証プロファイル（個人用と仕事用）。
* 予測可能なフォールバックを伴うシンプルな`/model`選択。
* テキストモデルと画像対応モデルの明確な区別。

## 可能な方向性（高レベル）

* モデル選択をシンプルに保つ：`provider/model`とオプションのエイリアス。
* プロバイダーが明示的な順序で複数の認証プロファイルを持つことを許可。
* すべてのセッションが一貫してフォールバックするように、グローバルなフォールバックリストを使用。
* 明示的に設定されている場合のみ画像ルーティングをオーバーライド。
