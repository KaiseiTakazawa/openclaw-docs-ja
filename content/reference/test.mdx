# テスト

* フルテストキット（スイート、ライブ、Docker）： [Testing](/help/testing)

* `pnpm test:force`: デフォルトの制御ポートを保持している残存するGatewayプロセスを終了させ、その後、サーバーテストが実行中のインスタンスと衝突しないように、孤立したGatewayポートでフルVitestスイートを実行します。以前のGatewayの実行でポート18789が占有されている場合に使用してください。

* `pnpm test:coverage`: V8カバレッジ（`vitest.unit.config.ts`経由）を使用してユニットスイートを実行します。グローバルしきい値は70%の行/ブランチ/関数/ステートメントです。カバレッジは、ユニットテスト可能なロジックに焦点を当てるために、統合重視のエントリポイント（CLIワイヤリング、Gateway/Telegramブリッジ、Webチャット静的サーバー）を除外します。

* Node 24+での`pnpm test`: OpenClawはVitestの`vmForks`を自動的に無効にし、`forks`を使用して`ERR_VM_MODULE_LINK_FAILURE` / `module is already linked`を回避します。この動作を強制するには、`OPENCLAW_TEST_VM_FORKS=0|1`を使用できます。

* `pnpm test:e2e`: Gatewayのエンドツーエンドスモークテスト（マルチインスタンスWS/HTTP/nodeペアリング）を実行します。デフォルトは`vmForks` + `vitest.e2e.config.ts`の適応ワーカーです；`OPENCLAW_E2E_WORKERS=<n>`で調整し、詳細なログを得るには`OPENCLAW_E2E_VERBOSE=1`を設定します。

* `pnpm test:live`: プロバイダーのライブテスト（minimax/zai）を実行します。APIキーと`LIVE=1`（またはプロバイダー特有の`*_LIVE_TEST=1`）が必要です。

## ローカルPRゲート

ローカルPRのランド/ゲートチェックを実行するには、以下を実行します：

* `pnpm check`
* `pnpm build`
* `pnpm test`
* `pnpm check:docs`

もし`pnpm test`が負荷のかかったホストでフレークした場合、回帰と見なす前に一度再実行し、その後`pnpm vitest run <path/to/test>`で隔離します。メモリ制約のあるホストでは、以下を使用します：

* `OPENCLAW_TEST_PROFILE=low OPENCLAW_TEST_SERIAL_GATEWAY=1 pnpm test`

## モデルレイテンシベンチ（ローカルキー）

スクリプト: [`scripts/bench-model.ts`](https://github.com/openclaw/openclaw/blob/main/scripts/bench-model.ts)

使用法：

* `source ~/.profile && pnpm tsx scripts/bench-model.ts --runs 10`
* オプション環境変数: `MINIMAX_API_KEY`, `MINIMAX_BASE_URL`, `MINIMAX_MODEL`, `ANTHROPIC_API_KEY`
* デフォルトプロンプト: “一言で返答してください: ok. 句読点や余分なテキストは不要です。”

最後の実行（2025-12-31、20回実行）：

* minimaxの中央値1279ms（最小1114、最大2431）
* opusの中央値2454ms（最小1224、最大3170）

## オンボーディングE2E（Docker）

Dockerはオプションですが、コンテナ化されたオンボーディングスモークテストには必要です。

クリーンなLinuxコンテナでのフルコールドスタートフロー：

```bash
scripts/e2e/onboard-docker.sh
```

このスクリプトは擬似ttyを介してインタラクティブウィザードを駆動し、設定/ワークスペース/セッションファイルを検証し、その後Gatewayを起動して`openclaw health`を実行します。

## QRインポートスモーク（Docker）

Node 22+で`qrcode-terminal`がDocker内で読み込まれることを確認します：
