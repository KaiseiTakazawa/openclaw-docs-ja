# プロンプトキャッシング

# Prompt caching

プロンプトキャッシングとは、モデルプロバイダーが変更されていないプロンプトプレフィックス（通常はシステム/開発者の指示やその他の安定したコンテキスト）をターン間で再利用できることを意味します。これにより、毎回再処理する必要がなくなります。最初の一致するリクエストがキャッシュトークンを記録し（`cacheWrite`）、その後の一致するリクエストはそれを読み取ることができます（`cacheRead`）。

これが重要な理由：トークンコストの削減、応答速度の向上、長時間実行されるセッションにおけるパフォーマンスの予測可能性の向上です。キャッシングがない場合、繰り返しのプロンプトは、ほとんどの入力が変更されていない場合でも、毎回フルプロンプトコストを支払う必要があります。

このページでは、プロンプトの再利用とトークンコストに影響を与えるすべてのキャッシュ関連の設定について説明します。

Anthropicの価格詳細については、以下を参照してください：
[https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

## 主な設定

### `cacheRetention`（モデルおよびエージェントごと）

モデルパラメータに対してキャッシュ保持を設定します：

```yaml
agents:
  defaults:
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "short" # none | short | long
```

エージェントごとのオーバーライド：

```yaml
agents:
  list:
    - id: "alerts"
      params:
        cacheRetention: "none"
```

設定のマージ順序：

1. `agents.defaults.models["provider/model"].params`
2. `agents.list[].params`（一致するエージェントID; キーによるオーバーライド）

### レガシー `cacheControlTtl`

レガシー値は引き続き受け入れられ、マッピングされます：

* `5m` -> `short`
* `1h` -> `long`

新しい設定には `cacheRetention` を推奨します。

### `contextPruning.mode: "cache-ttl"`

キャッシュTTLウィンドウの後に古いツール結果コンテキストをプルーニングし、アイドル後のリクエストが過剰な履歴を再キャッシュしないようにします。

```yaml
agents:
  defaults:
    contextPruning:
      mode: "cache-ttl"
      ttl: "1h"
```

完全な動作については、[Session Pruning](/concepts/session-pruning)を参照してください。

### ハートビートのウォーム保持

ハートビートはキャッシュウィンドウを温かく保ち、アイドルギャップ後の繰り返しキャッシュ書き込みを減少させることができます。

```yaml
agents:
  defaults:
    heartbeat:
      every: "55m"
```

エージェントごとのハートビートは `agents.list[].heartbeat` でサポートされています。

## プロバイダーの動作

### Anthropic（直接API）

* `cacheRetention` がサポートされています。
* Anthropic APIキー認証プロファイルを使用すると、OpenClawは未設定の場合にAnthropicモデル参照に対して `cacheRetention: "short"` を設定します。

### Amazon Bedrock

* Anthropic Claudeモデル参照（`amazon-bedrock/*anthropic.claude*`）は明示的な `cacheRetention` パススルーをサポートします。
* 非AnthropicのBedrockモデルは、ランタイムで `cacheRetention: "none"` に強制されます。

### OpenRouter Anthropicモデル

`openrouter/anthropic/*` モデル参照について、OpenClawはプロンプトキャッシュの再利用を改善するためにシステム/開発者プロンプトブロックにAnthropic `cache_control` を注入します。

### その他のプロバイダー

プロバイダーがこのキャッシュモードをサポートしていない場合、`cacheRetention` は効果がありません。

## チューニングパターン

### 混合トラフィック（推奨デフォルト）

メインエージェントに長寿命のベースラインを維持し、バースト通知エージェントではキャッシングを無効にします：

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
  list:
    - id: "research"
      default: true
      heartbeat:
        every: "55m"
    - id: "alerts"
      params:
        cacheRetention: "none"
```

### コスト優先のベースライン

* ベースライン `cacheRetention: "short"` を設定します。
* `contextPruning.mode: "cache-ttl"` を有効にします。
* ウォームキャッシュの恩恵を受けるエージェントのみ、ハートビートをTTL以下に保ちます。

## キャッシュ診断

OpenClawは埋め込まれたエージェント実行のための専用のキャッシュトレース診断を公開します。

### `diagnostics.cacheTrace` 設定

```yaml
diagnostics:
  cacheTrace:
    enabled: true
    filePath: "~/.openclaw/logs/cache-trace.jsonl" # オプション
    includeMessages: false # デフォルトはtrue
    includePrompt: false # デフォルトはtrue
    includeSystem: false # デフォルトはtrue
```

デフォルト：

* `filePath`: `$OPENCLAW_STATE_DIR/logs/cache-trace.jsonl`
* `includeMessages`: `true`
* `includePrompt`: `true`
* `includeSystem`: `true`

### 環境トグル（ワンオフデバッグ）

* `OPENCLAW_CACHE_TRACE=1` はキャッシュトレースを有効にします。
* `OPENCLAW_CACHE_TRACE_FILE=/path/to/cache-trace.jsonl` は出力パスをオーバーライドします。
* `OPENCLAW_CACHE_TRACE_MESSAGES=0|1` はフルメッセージペイロードキャプチャを切り替えます。
* `OPENCLAW_CACHE_TRACE_PROMPT=0|1` はプロンプトテキストキャプチャを切り替えます。
* `OPENCLAW_CACHE_TRACE_SYSTEM=0|1` はシステムプロンプトキャプチャを切り替えます。

### 検査すべきこと

* キャッシュトレースイベントはJSONL形式で、`session:loaded`、`prompt:before`、`stream:context`、`session:after`などのステージされたスナップショットを含みます。
* 各ターンのキャッシュトークンの影響は、通常の使用表面を通じて `cacheRead` と `cacheWrite` によって可視化されます（例えば `/usage full` やセッション使用の概要）。

## クイックトラブルシューティング

* ほとんどのターンで高い `cacheWrite`：揮発性のシステムプロンプト入力を確認し、モデル/プロバイダーがキャッシュ設定をサポートしていることを確認してください。
* `cacheRetention` が効果を持たない：モデルキーが `agents.defaults.models["provider/model"]` と一致していることを確認してください。
* キャッシュ設定を持つBedrock Nova/Mistralリクエスト：ランタイムで `none` に強制されることが予想されます。
