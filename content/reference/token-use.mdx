# トークン使用量とコスト

# Token use & costs

OpenClawは**トークン**を追跡しますが、文字数ではありません。トークンはモデル固有ですが、ほとんどのOpenAIスタイルのモデルは英語のテキストで約4文字を1トークンとして平均しています。

## システムプロンプトの構築方法

OpenClawは毎回独自のシステムプロンプトを組み立てます。これには以下が含まれます：

* ツールリスト + 短い説明
* スキルリスト（メタデータのみ；指示は`read`で必要に応じて読み込まれます）
* 自己更新指示
* ワークスペース + ブートストラップファイル（新しい場合は`AGENTS.md`、`SOUL.md`、`TOOLS.md`、`IDENTITY.md`、`USER.md`、`HEARTBEAT.md`、`BOOTSTRAP.md`、さらに存在する場合は`MEMORY.md`および/または`memory.md`）。大きなファイルは`agents.defaults.bootstrapMaxChars`（デフォルト：20000）によって切り捨てられ、総ブートストラップ注入は`agents.defaults.bootstrapTotalMaxChars`（デフォルト：150000）で制限されます。`memory/*.md`ファイルはメモリツールを介して必要に応じて使用され、自動的には注入されません。
* 時間（UTC + ユーザーのタイムゾーン）
* 応答タグ + ハートビートの動作
* ランタイムメタデータ（ホスト/OS/モデル/思考）

完全な内訳は[システムプロンプト](/concepts/system-prompt)を参照してください。

## コンテキストウィンドウでカウントされるもの

モデルが受け取るすべてのものがコンテキスト制限にカウントされます：

* システムプロンプト（上記のすべてのセクション）
* 会話履歴（ユーザー + アシスタントメッセージ）
* ツール呼び出しとツール結果
* 添付ファイル/トランスクリプト（画像、音声、ファイル）
* 圧縮要約とプルーニングアーティファクト
* プロバイダラッパーまたは安全ヘッダー（見えませんが、カウントされます）

画像については、OpenClawはプロバイダ呼び出しの前にトランスクリプト/ツール画像ペイロードをダウンサイジングします。これを調整するには`agents.defaults.imageMaxDimensionPx`（デフォルト：`1200`）を使用します：

* 低い値は通常、ビジョントークンの使用量とペイロードサイズを減少させます。
* 高い値はOCR/UI重視のスクリーンショットの視覚的詳細をより多く保持します。

実際の内訳（注入されたファイル、ツール、スキル、システムプロンプトのサイズごと）を確認するには、`/context list`または`/context detail`を使用します。詳細は[コンテキスト](/concepts/context)を参照してください。

## 現在のトークン使用量を確認する方法

チャットでこれを使用します：

* `/status` → **絵文字が豊富なステータスカード**がセッションモデル、コンテキスト使用量、最後の応答の入力/出力トークン、そして**推定コスト**（APIキーのみ）を表示します。
* `/usage off|tokens|full` → 各応答に**応答ごとの使用量フッター**を追加します。
  * セッションごとに持続します（`responseUsage`として保存されます）。
  * OAuth認証は**コストを隠します**（トークンのみ）。
* `/usage cost` → OpenClawセッションログからのローカルコストの概要を表示します。

他のインターフェース：

* **TUI/Web TUI:** `/status` + `/usage`がサポートされています。
* **CLI:** `openclaw status --usage`および`openclaw channels list`はプロバイダのクォータウィンドウを表示します（応答ごとのコストではありません）。

## コスト推定（表示される場合）

コストはモデルの価格設定構成から推定されます：

```
models.providers.<provider>.models[].cost
```

これらは`input`、`output`、`cacheRead`、および`cacheWrite`の**1MトークンあたりのUSD**です。価格設定が欠けている場合、OpenClawはトークンのみを表示します。OAuthトークンは決してドルコストを表示しません。

## キャッシュTTLとプルーニングの影響

プロバイダプロンプトのキャッシングはキャッシュTTLウィンドウ内でのみ適用されます。OpenClawはオプションで**キャッシュTTLプルーニング**を実行できます：キャッシュTTLが期限切れになった後、セッションをプルーニングし、その後キャッシュウィンドウをリセットして、次のリクエストが新しくキャッシュされたコンテキストを再利用できるようにします。これにより、セッションがTTLを超えてアイドル状態になるときのキャッシュ書き込みコストが低く抑えられます。

これを[ゲートウェイ設定](/gateway/configuration)で構成し、[セッションプルーニング](/concepts/session-pruning)で動作の詳細を確認してください。

ハートビートはアイドルギャップを越えてキャッシュを**温かく**保つことができます。モデルのキャッシュTTLが`1h`の場合、その直下にハートビート間隔を設定する（例：`55m`）ことで、フルプロンプトの再キャッシュを避け、キャッシュ書き込みコストを削減できます。

マルチエージェントセットアップでは、1つの共有モデル設定を維持し、`agents.list[].params.cacheRetention`でエージェントごとにキャッシュ動作を調整できます。

詳細なノブごとのガイドについては、[プロンプトキャッシング](/reference/prompt-caching)を参照してください。

Anthropic APIの価格設定では、キャッシュ読み取りは入力トークンよりも大幅に安価であり、キャッシュ書き込みは高い倍率で請求されます。最新の料金とTTL倍率については、Anthropicのプロンプトキャッシング価格を参照してください：
[https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

### 例：ハートビートで1時間キャッシュを温かく保つ

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
    heartbeat:
      every: "55m"
```

### 例：エージェントごとのキャッシュ戦略を持つ混合トラフィック

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long" # ほとんどのエージェントのデフォルトベースライン
  list:
    - id: "research"
      default: true
      heartbeat:
        every: "55m" # 深いセッションのために長いキャッシュを温かく保つ
    - id: "alerts"
      params:
        cacheRetention: "none" # 突発的な通知のためにキャッシュ書き込みを避ける
```

`agents.list[].params`は選択されたモデルの`params`の上にマージされるため、`cacheRetention`のみをオーバーライドし、他のモデルのデフォルトを変更せずに継承できます。

### 例：Anthropic 1Mコンテキストベータヘッダーを有効にする

Anthropicの1Mコンテキストウィンドウは現在ベータ版です。OpenClawは、サポートされているOpusまたはSonnetモデルで`context1m`を有効にすると、必要な`anthropic-beta`値を注入できます。

```yaml
agents:
  defaults:
    models:
      "anthropic/claude-opus-4-6":
        params:
          context1m: true
```

これはAnthropicの`context-1m-2025-08-07`ベータヘッダーにマッピングされます。

AnthropicをOAuth/サブスクリプショントークン（`sk-ant-oat-*`）で認証すると、OpenClawは`context-1m-*`ベータヘッダーをスキップします。なぜなら、Anthropicは現在その組み合わせをHTTP 401で拒否しているからです。

## トークン圧力を減らすためのヒント

* `/compact`を使用して長いセッションを要約します。
* ワークフロー内で大きなツール出力を削減します。
* スクリーンショットが多いセッションでは`agents.defaults.imageMaxDimensionPx`を低く設定します。
* スキルの説明を短く保ちます（スキルリストはプロンプトに注入されます）。
* 冗長で探索的な作業には小さなモデルを好みます。

正確なスキルリストのオーバーヘッドの公式については[スキル](/tools/skills)を参照してください。
