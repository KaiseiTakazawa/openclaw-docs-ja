# Railwayへのデプロイ

OpenClawをRailwayにワンクリックテンプレートでデプロイし、ブラウザで設定を完了します。  
これは「サーバー上でターミナルを使用しない」最も簡単な方法です：RailwayがGatewayを実行し、すべてを`/setup`ウェブウィザードを通じて設定します。

## クイックチェックリスト（新規ユーザー向け）

1. **Railwayにデプロイ**をクリックします（下記）。
2. `/data`にマウントされた**ボリューム**を追加します。
3. 必要な**変数**を設定します（少なくとも`SETUP_PASSWORD`）。
4. ポート`8080`で**HTTPプロキシ**を有効にします。
5. `https://<your-railway-domain>/setup`を開き、ウィザードを完了します。

## ワンクリックデプロイ

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Railwayにデプロイ
</a>

デプロイ後、**Railway → あなたのサービス → 設定 → ドメイン**で公開URLを見つけます。

Railwayは次のいずれかを行います：

* 生成されたドメインを提供します（通常は`https://<something>.up.railway.app`）、または
* 付けたカスタムドメインを使用します。

次を開きます：

* `https://<your-railway-domain>/setup` — セットアップウィザード（パスワード保護あり）
* `https://<your-railway-domain>/openclaw` — コントロールUI

## 取得できるもの

* ホスティングされたOpenClaw Gateway + コントロールUI
* `/setup`でのウェブセットアップウィザード（ターミナルコマンド不要）
* Railwayボリューム（`/data`）を介した永続ストレージにより、設定/認証情報/ワークスペースが再デプロイ後も生き残ります
* 後でRailwayから移行するためのバックアップエクスポートが`/setup/export`にあります

## 必要なRailway設定

### 公開ネットワーキング

サービスのために**HTTPプロキシ**を有効にします。

* ポート: `8080`

### ボリューム（必須）

次のようにマウントされたボリュームを添付します：

* `/data`

### 変数

サービスにこれらの変数を設定します：

* `SETUP_PASSWORD`（必須）
* `PORT=8080`（必須 — 公開ネットワーキングのポートと一致する必要があります）
* `OPENCLAW_STATE_DIR=/data/.openclaw`（推奨）
* `OPENCLAW_WORKSPACE_DIR=/data/workspace`（推奨）
* `OPENCLAW_GATEWAY_TOKEN`（推奨；管理者の秘密として扱う）

## セットアップフロー

1. `https://<your-railway-domain>/setup`にアクセスし、`SETUP_PASSWORD`を入力します。
2. モデル/認証プロバイダーを選択し、キーを貼り付けます。
3. （オプション）Telegram/Discord/Slackトークンを追加します。
4. **セットアップを実行**をクリックします。

TelegramのDMがペアリングに設定されている場合、セットアップウィザードはペアリングコードを承認できます。

## チャットトークンの取得

### Telegramボットトークン

1. Telegramで`@BotFather`にメッセージを送信します。
2. `/newbot`を実行します。
3. トークンをコピーします（`123456789:AA...`のように見えます）。
4. `/setup`に貼り付けます。

### Discordボットトークン

1. [https://discord.com/developers/applications](https://discord.com/developers/applications)にアクセスします。
2. **新しいアプリケーション** → 名前を選択します。
3. **ボット** → **ボットを追加**を選択します。
4. ボット → 特権ゲートウェイインテントの下で**MESSAGE CONTENT INTENTを有効にします**（必須、さもなくばボットは起動時にクラッシュします）。
5. **ボットトークン**をコピーし、`/setup`に貼り付けます。
6. ボットをサーバーに招待します（OAuth2 URLジェネレーター；スコープ：`bot`、`applications.commands`）。

## バックアップと移行

バックアップをダウンロードします：

* `https://<your-railway-domain>/setup/export`

これにより、OpenClawの状態とワークスペースがエクスポートされ、設定やメモリを失うことなく別のホストに移行できます。
