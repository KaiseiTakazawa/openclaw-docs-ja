# ペアリング
「ペアリング」は、OpenClawの明示的な**オーナー承認**ステップです。
これは二つの場所で使用されます：
1. **DMペアリング**（ボットと話すことが許可されている人）
2. **ノードペアリング**（どのデバイス/ノードがゲートウェイネットワークに参加できるか）
セキュリティコンテキスト: [セキュリティ](/gateway/security)
## 1) DMペアリング（インバウンドチャットアクセス）
チャネルがDMポリシー`pairing`で設定されている場合、未知の送信者は短いコードを受け取り、そのメッセージは**処理されません**まで承認されるまで。
デフォルトのDMポリシーは以下に文書化されています: [セキュリティ](/gateway/security)
ペアリングコード：
* 8文字、大文字、曖昧な文字なし（`0O1I`）。
* **1時間後に期限切れ**。ボットは新しいリクエストが作成されたときのみペアリングメッセージを送信します（おおよそ1時間に1回、送信者ごと）。
* 保留中のDMペアリングリクエストはデフォルトで**チャネルごとに3件**に制限されており、追加のリクエストは1件が期限切れまたは承認されるまで無視されます。
### 送信者を承認する
```bash
openclaw pairing list telegram
openclaw pairing approve telegram ```` ```
サポートされているチャネル: `telegram`, `whatsapp`, `signal`, `imessage`, `discord`, `slack`, `feishu`。
### ステータスの保存場所
`~/.openclaw/credentials/`に保存されています：
* 保留中のリクエスト: `-pairing.json`
* 承認された許可リストストア：
* デフォルトアカウント: `-allowFrom.json`
* 非デフォルトアカウント: `--allowFrom.json`
アカウントスコープの動作：
* 非デフォルトアカウントは、自分のスコープされた許可リストファイルのみを読み書きします。
* デフォルトアカウントは、チャネルスコープされた非スコープ許可リストファイルを使用します。
これらは機密情報として扱ってください（あなたのアシスタントへのアクセスを制限します）。
## 2) ノードデバイスペアリング（iOS/Android/macOS/ヘッドレスノード）
ノードは、`role: node`として**デバイス**としてゲートウェイに接続します。ゲートウェイは、承認が必要なデバイスペアリングリクエストを作成します。
### Telegram経由でペアリング（iOSに推奨）
`device-pair`プラグインを使用している場合、初回のデバイスペアリングを完全にTelegramから行うことができます：
1. Telegramでボットにメッセージを送信: `/pair`
2. ボットは二つのメッセージで応答します: 指示メッセージと別の**セットアップコード**メッセージ（Telegramで簡単にコピー/ペースト可能）。
3. あなたの電話で、OpenClaw iOSアプリを開く → 設定 → ゲートウェイ。
4. セットアップコードを貼り付けて接続します。
5. Telegramに戻って: `/pair approve`
セットアップコードは、次の内容を含むbase64エンコードされたJSONペイロードです：
* `url`: ゲートウェイWebSocket URL（`ws://...`または`wss://...`）
* `token`: 短命のペアリングトークン
セットアップコードが有効な間は、パスワードのように扱ってください。
### ノードデバイスを承認する
```bash
openclaw devices list
openclaw devices approve 
openclaw devices reject 
```
### ノードペアリングステートストレージ
`~/.openclaw/devices/`に保存されています：
* `pending.json`（短命; 保留中のリクエストは期限切れ）
* `paired.json`（ペアリングされたデバイス + トークン）
### 注意事項
* 従来の`node.pair.*` API（CLI: `openclaw nodes pending/approve`）は、別のゲートウェイ所有のペアリングストアです。WSノードは依然としてデバイスペアリングを必要とします。
