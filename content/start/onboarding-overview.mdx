# オンボーディング概要

OpenClawは、Gatewayがどこで実行されているか、またはプロバイダーの設定方法に応じて、複数のオンボーディングパスをサポートしています。

## オンボーディングパスを選択

* **CLIウィザード**：macOS、Linux、Windows（WSL2経由）用。
* **macOSアプリ**：AppleシリコンまたはIntel Macでのガイド付き初回実行用。

## CLIオンボーディングウィザード

ターミナルでウィザードを実行します：

```bash
openclaw onboard
```

Gateway、ワークスペース、チャネル、スキルの完全な制御を望む場合は、CLIウィザードを使用してください。ドキュメント：

* [オンボーディングウィザード (CLI)](/start/wizard)
* [`openclaw onboard` コマンド](/cli/onboard)

## macOSアプリオンボーディング

macOSで完全にガイドされたセットアップを希望する場合は、OpenClawアプリを使用してください。ドキュメント：

* [オンボーディング (macOSアプリ)](/start/onboarding)

## カスタムプロバイダー

リストにないエンドポイントが必要な場合、標準のOpenAIまたはAnthropic APIを公開するホスティングプロバイダーを含めて、CLIウィザードで**カスタムプロバイダー**を選択してください。以下の情報を求められます：

* OpenAI互換、Anthropic互換、または**不明**（自動検出）を選択。
* 基本URLとAPIキー（プロバイダーによって必要な場合）を入力。
* モデルIDとオプションのエイリアスを提供。
* 複数のカスタムエンドポイントが共存できるようにエンドポイントIDを選択。

詳細な手順については、上記のCLIオンボーディングドキュメントに従ってください。
