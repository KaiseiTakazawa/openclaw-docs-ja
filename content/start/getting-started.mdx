# はじめに

# はじめに

目標: 最小限のセットアップでゼロから最初の動作するチャットを作成すること。

  最速のチャット: コントロールUIを開く（チャンネルのセットアップは不要）。`openclaw dashboard`を実行してブラウザでチャットするか、ゲートウェイホストで`http://127.0.0.1:18789/`を開きます。
  ドキュメント: [ダッシュボード](/web/dashboard) と [コントロールUI](/web/control-ui)。

## 前提条件

* Node 22 以上

  不明な場合は、`node --version`でNodeのバージョンを確認してください。

## クイックセットアップ (CLI)

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

<img src="https://mintcdn.com/clawdhub/A8OxQpxR3DcyCCHY/assets/install-script.svg?fit=max&auto=format&n=A8OxQpxR3DcyCCHY&q=85&s=461cd845ecef4a1e5bda13a958ab6eac" alt="インストールスクリプトのプロセス" className="rounded-lg" width="1370" height="581" />

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

他のインストール方法と要件: [インストール](/install)。

```bash
openclaw onboard --install-daemon
```

ウィザードが認証、ゲートウェイ設定、およびオプションのチャンネルを構成します。詳細については[オンボーディングウィザード](/start/wizard)を参照してください。

サービスをインストールした場合、すでに実行中であるはずです：

```bash
openclaw gateway status
```

```bash
openclaw dashboard
```

コントロールUIが読み込まれれば、あなたのGatewayは使用可能です。

## オプションのチェックと追加機能

クイックテストやトラブルシューティングに便利です。

```bash
openclaw gateway --port 18789
```

構成されたチャンネルが必要です。

```bash
openclaw message send --target +15555550123 --message "Hello from OpenClaw"
```

## 有用な環境変数

OpenClawをサービスアカウントとして実行する場合やカスタムの設定/状態の場所を指定したい場合：

* `OPENCLAW_HOME` は内部パス解決に使用されるホームディレクトリを設定します。
* `OPENCLAW_STATE_DIR` は状態ディレクトリをオーバーライドします。
* `OPENCLAW_CONFIG_PATH` は設定ファイルのパスをオーバーライドします。

完全な環境変数のリファレンス: [環境変数](/help/environment)。
